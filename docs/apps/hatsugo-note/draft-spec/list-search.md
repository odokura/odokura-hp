# 一覧の検索性・一覧性

## 背景・課題

登録が 100 件規模に増えると、**「何を登録したか」をリストから探すのが難しくなる**。

現行の一覧（`app/(tabs)/list.tsx`）は次のとおりで、規模が増えると破綻しやすい。

- まとめ単位は `word` 完全一致（`latestPerWord`）
- **ソートは 3 種のみ**（更新順 / 登録順 / あいうえお順）
- **検索ボックスがない**
- フラットな `FlatList` を上からスクロールするだけ

→ 「探す（検索性）」と「見渡す（一覧性）」の両面で改善する。

## 目標

- **検索性**: 探したい語に最短で届く。表記ゆれ（`ぱぱ`/`パパ`）でも見つかる。
- **一覧性**: 全体を見渡し、目的のあたりへ素早く飛べる。
- 既存のソート・タップ遷移の体験は壊さない。

## 方針

### 1. 検索ボックス（インクリメンタル）

- 一覧タブ上部に検索入力を置き、入力に応じて即時フィルタ。
- **照合対象**: surface（`word`）／ concept の `label` ／ グループに含まれる surface。
  - グループ横断で当たる（`おとうさん` で検索しても `パパ` グループが出る）。
  - **memo は照合対象に含めない**（ノイズを避け、語そのものの検索に絞る）。
- **正規化して照合**: クエリと対象の両方を [dedup の正規化](./word-grouping/dedup-levels.md)で寄せる。
  - `normalizeKey`（全角半角・大小・空白）で大小・幅差を吸収。
  - さらに `foldForSuggestion`（カナ折り・長音）で `ぱぱ` クエリが `パパ` に当たる。
  - 部分一致（含む）で拾う。

### 2. 一覧性（見渡し・ジャンプ）

- **あいうえお順のとき、見出し（セクション）を出す**: `あ行 / か行 …`（または頭文字）でグルーピングし、sticky ヘッダーにする。
- **索引レール（MVP に含める）**: 画面端に `あ か さ た … A〜Z #` のジャンプ索引を置き、タップでそのセクションへスクロール（連絡先アプリ風）。
- **件数表示**: ヘッダーに総件数、各セクションに件数。
- 更新順 / 登録順のときはセクションなし（時系列の流れを保つ）。索引レールはあいうえお順のときだけ表示する。

### 3. 集約との相乗

- [単語の集約](./word-grouping/index.md)で行数自体が減る（`パパ`＋`おとうさん`＝1 行）→ 一覧性が直接改善。
- 検索はグループ内の全 surface を対象にするので、まとめても「どの言い方でも見つかる」。

## 読み（yomi）の扱いと限界

- あいうえお見出し・索引レールは **見出しラベルの先頭文字**で振り分ける。
- 漢字・ローマ字など読みが取れない語は、頭文字の字種で `A〜Z` / `#（その他）` バケツに入れる（完全な五十音整列はしない）。
- 幼児語の多くは仮名なので大半は自然に並ぶ。**読み（yomi）を持って正確に整列するのは後続**（concept/lemma 拡張と合わせて検討）。

## MVP スコープ

- 検索ボックス（正規化・部分一致・グループ横断、memo は対象外）
- あいうえおソート時のセクション見出し＋件数
- **索引レール**（あいうえお順のときのジャンプ）

後続: 日付範囲などの絞り込み、yomi による正確な整列、検索ヒットのハイライト。

## 一覧性の MVP 決定

- **セクション粒度** → 決定: **行単位**（`あ / か / さ …`）でセクション分け。索引レールも行単位。
- **索引レールの常時表示** → 決定: 50 音（行）＋ `A〜Z` ＋ `#` を**常に固定表示**し、該当語がない頭文字は淡色・タップ無効。
- **レールの並び** → 決定: かな（あ→わ）→ `A〜Z` → `#`（漢字・その他）の順。
