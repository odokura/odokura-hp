# 集約 UX（いつ・どう束ねるか）

入力負担を増やさない原則のもと、束ねる経路を 3 つに分ける。

## 1. 入力時（既定は何も聞かない）

- 基本は surface のみ入力。concept は聞かない。
- 例外として、既存に **L1（近似一致＝表記ゆれ）／L2（lemma 語形）／L3（concept・curated 辞書一致）** の候補があれば、任意の補助提案を出す（→ [重複判定基準](./dedup-levels.md)）。
  - 例: 「`おとうさん` を `パパ` と同じ意味としてまとめますか？」→ はい／いいえ／あとで。**粒度はユーザーの回答で決まる**。
  - なお **L0′（正規化一致）** は明らかなので確認せず黙ってまとまる（提案は出さない）。
- **対象は単語1件登録だけでなく、二語文（`multi_word`）の分割語それぞれにも同じ提案チェックを行う**。二語文は半角／全角スペースで分割した各語を個別レコードとして保存するため（→ [発語記録モデル](../record-model.md#入力と二語文の分割libwordts)）、分割後の各語に対して単語登録と同じ手順で L1/L2/L3 を判定し、該当すれば提案する。例えば「わんわん きた」を登録した場合、分割語 `わんわん` が既存の `いぬ` concept と L3 一致すれば提案を出す（`きた` は対象外なら提案しない）。複数の分割語がそれぞれ候補を持つ場合は、語ごとに順番に確認する。
- **「はい」を選んだら、まとめ先の見出し名（一覧に表示されるラベル）を確認・編集できる入力欄を出す。** 既定値は提案ラベル（＝既存側の表記）だが、ユーザーが書き換えてから確定できる。これは[手動集約](#2-後から手動集約)の確認フローと同じ入力 UI を使う。「どちらの表記が一覧に残るか」を確定前に必ず見せる（Alert の文言だけで決めさせない）。
  - ただし、提案が**既に存在する concept への参加**（候補に既存の `conceptId` が付いている）場合は、見出し名は確定済みなので**見出し編集は省略**する。確認文言も新規グループ作成時とは分ける:
    - 新規（まだ concept が無い2語を初めて束ねる）: 「`おとうさん` を `パパ` とまとめられます。」→ はい／いいえ／あとで → はいなら見出し編集へ。
    - 既存（提案先に `conceptId` がある）: 「`いぬ` は既に `わんわん` としてまとめられています。同様にまとめますか？」→ はい／いいえ／あとで → はいなら見出し編集をスキップし、既存ラベルのまま `conceptId` を付与。
- 断れば従来どおり別単語のまま。**自動確定しない。**

## 2. 後から（手動集約）

- 単語詳細から、「この語を ◯◯ とまとめる／解除する」操作で `conceptId` を付与・解除できる（MVP は単語詳細に置く。専用「まとめ」画面は後段）。
- まとめを確定する前に、見出し名（ラベル）を確認・編集する入力欄を出す（既定値は提案ラベル）。**このラベル編集 UI は入力時の提案（→ 1）と共通**で、確認 UI を2系統に分けない。
- ただし**既に concept が存在するグループへの参加**の場合は、入力時の提案と同じ理由で見出し編集を省略する（確認文言も「既に ◯◯ としてまとめられています。同様にまとめますか？」に揃える）。
- 非破壊。いつでも解除でき、surface レコードは残る。

## 3. 補助提案の元（curated 辞書）

- 幼児語（赤ちゃん言葉）↔ 標準語の語族をカテゴリ単位でまとめた curated 辞書を持つ（実体は `lib/aggregation-suggest.ts` の `CURATED_GROUPS`）。
  - 家族: `ママ`/`おかあさん`/`お母さん`/`母`、`パパ`/`おとうさん`/`お父さん`/`父`、`じいじ`/`おじいちゃん`/`祖父`、`ばあば`/`おばあちゃん`/`祖母`、`にいに`/`おにいちゃん`/`兄`、`ねえね`/`おねえちゃん`/`姉`
  - 動物: `わんわん`/`いぬ`/`犬`、`にゃんにゃん`/`ねこ`/`猫`、`もーもー`/`うし`/`牛`、`ぶひぶひ`/`ぶた`/`豚`、`ぴよぴよ`/`ひよこ`/`とり`/`鳥`、`けろけろ`/`かえる`/`蛙`
  - 乗り物: `ブーブー`/`くるま`/`車`、`ぽっぽ`/`でんしゃ`/`電車`
  - 食事: `まんま`/`ごはん`/`ご飯`、`みるく`/`ミルク`/`牛乳`
  - 体: `あんよ`/`あし`/`足`、`おてて`/`て`/`手`、`おめめ`/`め`/`目`、`おくち`/`くち`/`口`
  - 身の回り・場所: `くっく`/`くつ`/`靴`、`おんも`/`そと`/`外`
  - 動作・あいさつ: `ねんね`/`ねる`/`眠る`、`ばいばい`/`さようなら`
  - 個々の語の追加・調整は実装側（コードレビュー）で行い、ここではカテゴリの網羅性のみを仕様として管理する。
- 将来的に表現バリエーション（複数形・活用形）も視野に入れる（`car` / `cars`、`go` / `going`）。
