# 単語の集約

発語の「言い方（表現）」と「意味」が成長とともにずれていく問題に対し、surface（実際の発話）を失わずに、意味・基本形でまとめて見られる「集約」の仕組みを設計する。

## 背景と課題

発語は成長とともに、**「言い方（表現）」と「意味」の対応がずれていく**。

- 同じ意味を別の言い方で言うようになる
  - `パパ` → `おとうさん`
  - `わんわん` → `いぬ`
  - `ブーブー` → `くるま`
- 同じ語でも形が変わる（複数形・活用形・発音の揺れ）
  - `car` → `cars`
  - `go` → `going`
  - `ぱぱ` → `パパ`

現状は[発語記録モデル（現行仕様）](../record-model.md)のとおり、「実際に言ったことば（surface form）」単位でしか記録できない。グルーピングは `word` 文字列の完全一致のみ。そのため次の問題が起きる。

- **意味でまとめて並べ替え（ソート）できない**
- 同じ意味・同じ語が別レコードになり、**二重登録**になる

そこで、surface（実際の発話）を失わずに、意味や基本形で **まとめて見られる「集約」の仕組み** が必要になる。

## 目標

- 実際に言ったことばをそのまま残しつつ、意味・基本形でまとめて見られるようにする。
- 二重登録を避け、意味カテゴリ単位での振り返り・ソートを可能にする。
- 入力負担を増やさない（基本入力は surface のみ）。

## 非目標（この機能でやらないこと）

- 大規模辞書の導入
- 自動的な同義語推定
- 複数形・活用形・幼児語の自動正規化を必須にすること
- concept（意味のまとまり）の入力を必須にすること
- **既存レコードの破壊的なマージ**（surface を失う統合はしない）

家庭差・個人差・多言語家庭の複雑さがあるため、辞書は「小さく始める」前提とする。
