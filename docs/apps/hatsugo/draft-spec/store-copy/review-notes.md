# Hatsu-go 審査・スクリーンショット・プライバシー

ストア審査に添える前提、スクリーンショットの文言、データ安全性の申告をまとめる。
表現は[概要・安全ライン](./index.md)に従い、診断・治療の効果を主張しない。

## スクリーンショット文言の例

| 画面 | 方向性 |
| --- | --- |
| 記録 | ことばを記録して、わが子の世界を見つめる |
| 記録入力 | タップで記録。動画・状況メモも一緒に残す |
| グラフ・サマリー | 月ごとに語彙・理解度を振り返る |
| 履歴一覧 | 語彙ごとの推移を後から探せる |
| Premium | バックアップ・データ移行・動画無制限 |

- 数値の「発達スコア」的な診断風の演出は使わない（記録であり測定・診断ではない）。

## 審査メモ（Review Notes）に入れる前提

- 医療アプリではない。発達の診断・判定は行わない。親が記録するための管理ツール。
- 発語データ・動画は外部サーバーにアップロードせず、端末内にのみ保存する。
- Premium は買い切り（サブスクではない）。無料で記録・グラフ化・履歴の基本機能を無制限に使える。
- Premium の価値はバックアップ、端末間・OS 間データ移行、動画投稿の無制限化。
- カメラ/写真は動画記録のために使う。目的をダイアログで説明する。

## データ安全性の申告

iOS の App Privacy と Google Play のデータ安全性で申告する内容をそろえる。

| 申告項目 | 内容 |
| --- | --- |
| データ収集 | 端末内にのみ保存。外部サーバーへアップロードしない |
| データ共有（第三者提供） | なし |
| 動画・写真 | 発語記録のために端末内に保存。外部送信しない |
| バックアップ・データ移行 | Premium 機能。暗号化処理で扱う |
| 位置情報・連絡先 | 収集しない |
| 購入情報 | ストア（Apple / Google）の課金基盤に準拠。レシートは端末内で扱う |

- iOS: App Privacy は端末内保存・第三者共有なしの実態に合わせて申告する。
- Google Play: データ安全性フォームで端末内保存・共有なしを申告する。
- バックアップ/移行で暗号化を用いる旨を審査メモに補足する。

## 年齢・レーティング

- 主対象は保護者。子ども向けカテゴリで申請するかは各ストアの方針に合わせて確定する。
- レーティング区分は各ストアの質問票の回答で確定する。

## 権限の説明文（ストア/OS ダイアログ）

- カメラ/写真: 「発語の様子を動画で記録するために使います。」
