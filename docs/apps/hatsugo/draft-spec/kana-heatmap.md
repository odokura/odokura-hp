# ホームの50音ヒートマップ

## 背景・目的

ホーム画面（`app/(tabs)/index.tsx`）は現在、サマリー・単語数の推移・最近の記録で構成される。発語の「量」（単語数とその推移）は見えるが、**「どの音が出てきているか」という音の面は見えない**。

- 登録された単語の**読みに含まれる音（モーラ）**を五十音表に割り付け、音ごとの単語数に応じてマスの背景色が明るくなるヒートマップを表示する
- **「出てきた音の広がり」**を一目で可視化し、記録が増えるほど表が点灯していく見せ方にする

### 設計判断（言語聴覚の観点でのレビュー結果）

- **「苦手な音」の可視化とはしない**。登録データは語彙リストであって発音の正確さは保証しない（子どもが「さかな」を「たかな」と発音していても、親は「さかな」と登録し得る）。また構音の獲得には正常な年齢幅があり（サ行・ラ行は 4〜6 歳が目安）、低年齢で暗いマスを「苦手」と示すと不要な不安や不適切な発音練習を誘発しかねない。strength ベース（点灯していく表）の見せ方に統一する。
- **先頭の音ではなく、読みに含まれる全モーラを数える**。語頭限定は 1 単語から 1 音しか取れずデータが薄い上、語頭に来やすい音の語彙的な偏り（る・ぬ 等で始まる幼児語はほぼ無い）をそのまま映してしまう。全モーラ集計は音インベントリ分析（independent analysis）の考え方に沿い、実装コストも同等。
- 発達目安や非診断であることの注記は**画面には出さない**。非診断であることはアプリ紹介ページに明記済みであり、画面内の注釈の追加は視認性を下げる。0 件マスの解釈に関する注記も同様に出さない。

## 配置

ホームの「単語数の推移」（`StackedBarChart`）の直下、「最近の記録」の上に置く。

表示は「単語数の推移」と同じ**カード形式**とする（`ThemedView`・padding 12・borderRadius 12・同じ影と elevation）。タイトル・表・対象外注記をすべてカード内に収め、タイトルの見た目もカード見出し（`type="subtitle"`・marginBottom 6）に合わせる。

## 表示仕様

### セクションタイトル

- **「音の広がり」**で確定。ja 専用機能のため訳語の検討は不要（→ [多言語方針](#多言語i18n方針)）。

### 表の構成とブロックレイアウト

レイアウト方針: **表が大きすぎることを最大のリスクとみなし、隙間を最小化した密な配置で画面幅に合わせて可変させる**。小さくなる方向は許容する（MVP はタップ操作のない静的表示のため、セルの小ささが操作性を損なうことはない）。

構成は次の 3 ブロックを 2 バンド・計 8 行に詰める（縦積み 3 ブロック＝13 行にしない）。

- **上段バンド（5 行・15 列）**: 右側に清音ブロック（10 列）、左側に濁音・半濁音ブロック（5 列）を横並び
- **下段バンド（3 行・11 列）**: 拗音ブロックを**グリッド全幅に引き伸ばして**配置する（空き列を作らない）

```text
（右から左へ読む・上段バンド 5 行＋下段バンド 3 行）

ぱ ば だ ざ が │ わ ら や ま は な た さ か あ
ぴ び ぢ じ ぎ │ を り ・ み ひ に ち し き い
ぷ ぶ づ ず ぐ │ ん る ゆ む ふ ぬ つ す く う
ぺ べ で ぜ げ │ ・ れ ・ め へ ね て せ け え
ぽ ぼ ど ぞ ご │ ・ ろ よ も ほ の と そ こ お

りゃ  みゃ  ぴゃ  びゃ  ひゃ  にゃ  ちゃ  じゃ  しゃ  ぎゃ  きゃ
りゅ  みゅ  ぴゅ  びゅ  ひゅ  にゅ  ちゅ  じゅ  しゅ  ぎゅ  きゅ
りょ  みょ  ぴょ  びょ  ひょ  にょ  ちょ  じょ  しょ  ぎょ  きょ
```

- **清音ブロック（46字）**: 子ども向けあいうえお表と同じく**右端があ行**の伝統配置。や行の い・え段は空白のまま残す（五十音表の形の認知を優先。行数が増えるわけではないので詰めない）。**わ列は わ・を・ん を上から詰めて配置**する（子ども向け表の慣習に合わせつつ、ん のための独立列を作らないため）。
- **濁音・半濁音ブロック（が・ざ・だ・ば・ぱ行、25マス）**: 清音からの続きとして右から が→ぱ の順に並べ、表全体が「右から左へ読み進む」一貫性を保つ。
  - 清音に寄せない理由: ぱ・ば行は発達的にむしろ早い音であり、清音マスに混ぜると可視化がゆがむ。
- **拗音ブロック（きゃ〜りゃ の 11 列 × ゃ・ゅ・ょ の 3 段、33マス）**: こちらも右から きゃ→りゃ の順。11 列をグリッド全幅に引き伸ばすため空きマスはない。セルは横長（幅＞高さ）になり、2 文字表示には都合がよい。列位置は上段バンドと揃わないが、バンド間ギャップで別ブロックとして区別できる。
  - きゃ を き に寄せない理由は濁音と同じ。拗音は獲得の遅い音群であり、基底かなのマスに混ぜると可視化がゆがむ。
- 行・段のラベルやブロック見出しのテキストは置かない（かな自体がラベルであり、隙間の最小化を優先）。ブロックの区切りはギャップのみで表す。

### サイズ計算（可変ルール）

- コンテンツ幅は**カード内側の実測幅**を使う（`StackedBarChart` と同じ onLayout 計測。目安: 画面幅 − ホーム左右 padding 32pt − カード padding 24pt）。
- 上段バンドのセルは正方形。一辺 s ＝（コンテンツ幅 − セル間ギャップ合計 − ブロック間ギャップ）÷ 15。
  - セル間ギャップ 2pt、ブロック間ギャップ（清音｜濁音の間、上下バンドの間とも）6pt。
- 拗音セルは高さ s・幅 w ＝（グリッド全幅 − 10 × 2pt）÷ 11 の横長（グリッド全幅 ＝ 15s ＋ 13 × 2pt ＋ 6pt）。
- **s の上限は 24pt**。上限に達したらグリッド全体を中央寄せする（タブレット・横向きで表が巨大化するのを防ぐ。大きすぎ回避を優先）。
- **下限は設けない**。横スクロールもさせない（常に画面幅に収める）。
- 文字サイズはセルに比例させる（目安 s × 0.55。拗音セルの 2 文字は s × 0.42〜0.5 目安。横長化で幅に余裕があるため実機で調整）。
- 目安: 幅 375pt の端末で内側幅 ≈ 319pt・s ≈ 19pt・拗音セル幅 ≈ 27pt、表全体の高さ ≈ 170pt。幅 320pt 級では s ≈ 15pt 前後まで縮み読みやすさは落ちるが、「大きすぎるより小さくなる方を許容する」方針どおり（既知の限界）。
- 画面回転・ウィンドウ幅の変化には onLayout の再計測で追従する。

### ヒートマップの色

- マスの背景色は「その音を含む発語済み単語数」の**離散5段階**: 0 / 1 / 2〜3 / 4〜6 / 7以上。
  - 全モーラ集計では 1 単語が複数マスに寄与するため、語頭のみの場合よりセルあたりの件数は増える。しきい値の妥当性は論点（下記）。
- 0 件は沈んだ無彩色（未点灯）。件数が増えるほど**明るく・鮮やかに**（音が「点灯していく」メタファー）。
- ライト／ダーク両テーマに対応し、全レベルでかな文字が読めるコントラストを保つ。具体的なカラー値は実装時に決める。

### セル内表示

- かな（拗音は 2 文字）のみ。件数の数字は出さない（色で表現する）。
- MVP ではタップ操作なし（静的表示）。

## 集計仕様

### 集計単位

- 記録数ではなく**単語（グループ）数**。ホーム・一覧と同じ `groupRecords` の結果を使う。
- 1 つの単語は、読みに含まれる**各モーラのマスへ 1 ずつ寄与する**。同一単語内に同じモーラが複数回あっても 1 回（例: わんわん → わ に 1・ん に 1）。
- 同じ単語を何度記録しても 1 カウント。[集約](./word-grouping/index.md)済みグループも 1 カウント。

### 発語済みの判定

- グループの最新レコードの `speechStatus` が `speech_only`（発語のみ）・`understood`（理解して話せる）・`multi_word`（二語文以上）のものを対象とする。
- **`understood_only`（理解のみ・未発語）は含めない**。表出された語のみを対象とするため。
- 「最新レコードのステータスで現状を判定する」考え方は推移グラフ（`StackedBarChart`）と同じ。

### 集計ソース（設定で切り替え）

どのテキストから音を拾うかを、設定「音の広がり」で**「単語」「読み」の 2 択**から切り替えられる。**既定は「読み」**。名称は記録入力フォームのフィールド名（単語／読み）に合わせる。

- **読み（既定）**: グループの読みを[検索・一覧性の索引と同じ優先順](./list-search.md#読みyomiの扱いと限界)で解決して使う: 保存された `yomi` → ラベル先頭のかな → curated 辞書ヒント → グループ内のかな surface。漢字表記の語も読みが取れれば表に乗る。
- **単語**: グループのラベル（代表表記）をそのまま使い、読みの推定・フォールバックは行わない。`yomi` は IME のキー入力からの自動推定で誤りが混じり得るため、登録した文字どおりの挙動を選びたい利用者向け。

### 音（モーラ）の分解

- 集計ソースのテキストを次の規則でモーラ列に分解し、各モーラをマスに割り付ける:
  - カタカナはひらがなに正規化する
  - **拗音（基底かな＋ゃ・ゅ・ょ）は 2 文字で 1 モーラとし、拗音ブロックの専用マスに計上する**（きゃべつ → きゃ・べ・つ）
  - 促音（っ）・長音（ー）は専用マスを設けず読み飛ばす
  - その他の小書き（ぁ 等）は基底文字に正規化する（データ異常時の保険）
  - かなに解決できない文字（英字・記号など）は読み飛ばす
  - 濁音・半濁音は清音に寄せず、独立マスに計上する
- 集計ソースのテキストにかなが 1 文字も含まれない語は集計対象外。

### 対象外の語の扱い

- 集計ソースからかなが得られない語は表に含めない（「読み」モードでは読み解決できない英語など、「単語」モードでは表記にかなを含まない漢字・英語表記の語）。
- 対象外が 1 語以上あるときのみ、表の下に 1 行の注記を出す: **「かなで数えられない N 語は含まれていません」**（「読みのわからない」ではなく、集計ソースが単語・読みのどちらでも成り立つモード非依存の文言にする）。

## 設定

- 設定画面の「アプリ設定」に**「音の広がり」項目を追加し、「単語」「読み」の 2 択**で切り替える。**既定は「読み」**。
- 設定項目はロケールが `ja` のときのみ表示する（→ 多言語方針）。
- `AppSettings` に集計ソースのフィールドを追加する（例: 'word' | 'yomi'）。既存ユーザーの保存済み設定には deep merge の後方互換で既定値（読み）が入る。
- 設定変更はホームの再フォーカス時の再集計で表に反映される（既存のホームのデータ読み込みフローに乗せる。専用の即時同期は作らない）。

## 多言語（i18n）方針

- **音の広がりは日本語（ja ロケール）専用の機能とする**。五十音は日本語固有の音体系であり、かな語彙を前提とする本機能をロケール横断にはしない。
- **ja 以外（en / zh-Hant）では、ホームのセクション・設定画面の項目とも完全非表示**。
  - 「かな解決できた単語が 1 語以上あれば表示する」というデータ駆動の表示条件は**採らない**。該当状況がほとんど発生しないため対応しない（決定）。
- `ja` では常に表示する（0 件でも「これから点いていく表」として見せる）。
- 言語設定を ja 以外へ切り替えると表・設定項目は消えるが、保存済みの集計ソース設定値は保持され、ja に戻せばそのまま効く。
- 文言キーは実装ルール（型）どおり 3 言語のファイルに置くが、**表示されるのは ja のみ**。en / zh-Hant の値は素直な直訳を入れておけばよく、訳語の検討・レビューは不要。
- セルのかなは UI 文言ではなく表示データであり、翻訳・ローマ字併記はしない。
- 各言語版の相当機能は作らない:
  - **英語の音素表は恒久的な非目標**。英語は綴りと音の対応が崩れており、発音辞書なしに表記から音を取れない。
  - **zh-Hant の注音（ㄅㄆㄇ）表**は音節頭子音のグリッドとして構造的には同型で作れるが、読み（注音）の取得手段がなく、[検索・一覧性の拼音索引バックログ](./list-search.md)と同じ扱いでバックログとする。
  - 将来の他言語対応を見越した先回りの抽象化（グリッド差し替え可能な汎用エンジン化）はしない。ja 固有で実装する。

### 文言（ja・確定）

- セクションタイトル・設定ラベル: 「音の広がり」
- 設定選択肢: 「単語」「読み」（記録フォームのフィールド名と同語）
- 対象外注記: 「かなで数えられない N 語は含まれていません」

## 実装方針

- 集計ロジック（読み解決の共用・モーラ分解・マス集計）は lib/ に純関数で置き（例: lib/kana-heatmap.ts）、テストを同階層に置く。
- 既存 `resolveGroupIndexKey`（`lib/index-key.ts`）は読み解決と行バケツ化が一体なので、読み解決部を分離して共用する。
- 表示コンポーネントは components/（例: components/kana-heatmap.tsx）。View の格子と `StyleSheet` で描けるため SVG・依存追加は不要。

## テスト観点

- 全モーラ集計（さかな → さ・か・な の 3 マスに寄与）
- 同一単語内の重複モーラが 1 回になる（わんわん → わ 1・ん 1）
- 拗音が独立マスに入り、基底かなに混ざらない（きゃべつ → きゃ・べ・つ。き には入らない）
- 濁音・半濁音が清音マスに混ざらない（ぱぱ → ぱ）
- 促音・長音・かな以外の文字の読み飛ばし（らっぱー → ら・ぱ）
- カタカナ・小書きの正規化
- `understood_only` が除外される
- グループ単位のカウント（同一単語の重複記録が 1 カウント）
- 読み解決のフォールバック順（yomi → ラベル → curated 辞書 → かな surface）
- 集計ソース設定の切り替え（「単語」で漢字表記の語が対象外になり、読み推定が使われない。「読み」で読み解決される）
- 設定の後方互換（保存済み設定に集計ソースが無くても既定「読み」になる）
- ロケールが `ja` 以外のとき、ホームのセクションと設定画面の項目がともに表示されない

## MVP スコープ

- 清音 46 ＋濁音・半濁音 25 ＋拗音 33 のヒートマップ表示（静的・タップなし）
- 全モーラ集計・5 段階の離散カラー・ライト／ダーク対応
- 集計ソース設定（単語／読み・既定は読み）と設定画面の項目追加
- 対象外語数の注記
- ja のみ表示（他ロケールは設定項目も含め完全非表示）

## 後続候補

- マスをタップ → その音を含む単語一覧へジャンプ
- 共有画像への掲載 → [共有画像](./share-image.md) で仕様化済み（移動）
- 「新しく出てきた音」のハイライト（月齢・期間と絡める）
- 調音の発達順序（パ行→カ行→サ行…）に沿った並べ替え・グルーピング表示

## 未確定・論点

- カラースケールの具体値としきい値（0 / 1 / 2〜3 / 4〜6 / 7以上）の妥当性
- ギャップ（2pt / 6pt）・セル上限（24pt）・文字比率（0.55）の定数は実機の見た目確認で微調整
