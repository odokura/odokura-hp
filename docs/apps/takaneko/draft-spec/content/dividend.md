# 配当（shareholder_return）

shareholder_return 種別のうち配当の仕様。**自己株式取得は XBRL 非対象（PDF 本文のみ）のため凍結し、
Phase 2 の text-rule で扱う**（→ [将来種別（凍結）](./future.md)）。
XBRL 読み取りの共通規則は[決算実績](./financial-result.md)に従う。

## 提供する価値

株主還元の額と対象期間、つまり企業が株主へ資本をどう配分するかを把握できる。
株主還元の良し悪しを表す評価を付けない。

## 受け入れる文書

| 経路 | 文書 | 扱い |
| --- | --- | --- |
| a | 決算短信サマリーの配当欄 | 当期実績・当期予想・来期予想の年間配当を候補化 |
| b | 配当予想の修正開示（「業績予想及び配当予想の修正」の配当部分を含む） | 修正後の年間配当を候補化。XBRL 添付がなければ保存のみ |
| c | 剰余金の配当（決議）開示 | XBRL 添付がある場合だけ（有無は要確認）、単一区分（中間・期末）の決定配当を候補化。なければ保存のみ |

いずれも対象銘柄集合内・XBRL 添付が条件。PDF 本文は解析しない。

## 抽出（要確認）

短信サマリーの配当欄は、年度（前期実績・当期実績または予想・来期予想）を行、四半期区分
（第 1〜第 3 四半期末・期末・合計）を列とする表に対応する XBRL 要素を持つ。要素名・コンテキストは
フィクスチャで確定する。論理規則:

| 項目 | 規則 |
| --- | --- |
| fiscal_year_end | 対象事業年度の期末年月日 |
| classification | 実績 / 予想。行（年度）と開示時点から決まる |
| annual_total | 年間合計の 1 株配当額（円銭。原文文字列と 1/100 円整数の両方で保存） |
| breakdown | 四半期区分ごとの 1 株額。属性として保存（本文には出さない） |
| record_date_type | 経路 c のみ。中間 / 期末 |

- 候補にするのは**年間合計が数値として確定している**ファクトだけとする。年間または一部区分が
  「未定」の年度は保存のみ（未定から決定に変わった時点で、新しい開示から候補になる）。
- 記念配当・特別配当の区分は属性として保存し、本文には出さない。
- 株式分割等による 1 株額の調整・換算はしない。開示の原文値だけを扱う。

## 初出と変更（ファクトの二形）

配当は「現在いくらか」だけでなく「いくらからいくらへ変わったか」を事実として識別する。

抽出した年間額は、同一の（対象年度、classification、fact_scope）で**表示済みの直近値**と比較する。

| 状況 | ファクト |
| --- | --- |
| 表示済みの値がない | **初出**ファクト（現在値） |
| 表示済みの直近値と同額 | ファクトを作らない（再掲。抑止理由 no_change で記録） |
| 表示済みの直近値と異なる | **変更**ファクト（修正前 → 修正後） |

- 変更ファクトの修正前値は、修正開示（経路 b）に前回値の記載があればそれを使い、なければ表示済みの
  直近値を使う。両者が矛盾する場合は開示記載を優先し、矛盾を operation_event に記録する。
- 修正前後の差額は保存する（本文には出さない）。
- classification が変わる遷移（予想 → 実績の確定）は、classification 別のキーになるため初出として扱う。

## 確定する企業ファクト

対象事業年度 × classification × fact_scope（annual = 年間合計 / interim / final = 経路 c の単区分）
ごとに、初出または変更の一ファクト。

## ローカル表示テンプレート

初出:

~~~text
<企業名>（<コード>）｜配当
<対象期>の<区分>配当は1株<金額>円
~~~

変更:

~~~text
<企業名>（<コード>）｜配当
<対象期>の<区分>配当を1株<修正前>円から<修正後>円に修正
~~~

| 表示 | 規則 |
| --- | --- |
| 対象期 | 「2026年3月期」（事業年度） |
| 区分 | 予想 / 実績 / 中間（決定） / 期末（決定） |
| 金額 | 円銭。小数第 2 位まで、末尾の 00 銭は省く（50 円、25.5 円、12.25 円） |

## 文書内の採用上限（サブ上限）

同一文書から採用する shareholder_return 候補は 1 件までとする。優先順は、変更を初出より先に、
同順位では来期予想を当期実績より先に採用する。これにより通期短信は典型的に配当 1 件 +
決算実績 2 件の計 3 件になる（→ [スコープと段階計画](../scope.md)の決算集中日の例）。
採用されなかった候補は抑止理由付きで保存する。

## 優先度と重複キー

- priority = 90。決算短信 1 通の中では配当が決算実績（80 台）より先に採用される。
- 表示重複キー:

~~~text
初出: company_id / shareholder_return / action_subtype=dividend / fact_scope /
      fiscal_year_end / classification
変更: 初出のキー + 修正前額 + 修正後額
~~~

- 初出キーに金額は含めない（同一対象の初出は一度だけ。再掲はファクト生成の段階で no_change 抑止）。
- 変更キーは修正前後の額を含むため、50 円 → 60 円 → 50 円の再修正でも（60, 50）が新キーになり
  表示される。
- 経路 c の決議と経路 a の年間合計は fact_scope が異なるため独立に投稿され得る（決定と年間計は
  別の事実として許容する）。
- 訂正開示は correction_kind = source のバイパス（共通規則）。

## 表示しない条件（抑止理由コード）

共通抑止に加えて:

| 理由コード | 条件 |
| --- | --- |
| no_xbrl | 経路 b・c で XBRL 添付がない |
| annual_undetermined | 年間合計が未定・非数値 |
| amount_form_invalid | 円銭が 1/100 円で表せない、負値 |
| row_mismatch | 年度行と四半期区分列の対応が取れない |
| no_change | 表示済みの直近値と同額の再掲 |
| kind_doc_cap | 文書内サブ上限（1 件）から漏れた候補 |

## フィクスチャ

| # | ケース | 確認すること |
| --- | --- | --- |
| 1 | 通期短信（当期実績 + 来期予想あり） | 2 候補からサブ上限で来期予想 1 件を採用。決算実績と合わせ 3 投稿 |
| 2 | 四半期短信で予想に変化なし | no_change 抑止 |
| 3 | 配当予想の修正開示 | 変更ファクト、遷移テンプレート、開示記載の前回値を使用 |
| 4 | 年間未定の会社 | annual_undetermined 抑止 |
| 5 | 未定から決定への遷移 | 初出として表示される |
| 6 | 剰余金の配当（決議、XBRL あり・なし各 1） | 単区分ファクト / no_xbrl 抑止 |
| 7 | 記念配当を含む会社 | 年間合計のみ本文、区分は属性保存 |
| 8 | 再修正で元の額に戻る（50 → 60 → 50） | 変更（60 → 50）が新キーで表示される |

受入基準は決算実績と同じ 4 条件。加えてフィクスチャ 1 のサブ上限選抜と、8 の再修正投稿を
結合テストで確認する。
