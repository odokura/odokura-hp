# 業績予想の修正（financial_forecast_revision）

XBRL の読み取り・値の正規化・根拠箇所の記録は[決算実績](./financial-result.md)の規則を共通で使い、
本ページは差分だけを定める。

## 提供する価値

会社が通期または対象期間の見通しを、何の指標について、どの値からどの値へ変更したかを把握できる。

## 受け入れる文書

修正前・修正後（前回予想・今回予想）を同じ表内で確認できる、業績予想の修正を明示する開示。

1. 銘柄コードが対象銘柄集合に含まれる。
2. 表題が「業績予想」と「修正」を含む（「業績予想及び配当予想の修正」を含む。特別損益の計上等と
   併記された複合表題も受け入れる）。
3. XBRL の添付がある。**添付がない予想修正開示（自由様式）は原本保存のみとし、投稿しない。**
   この規則により PDF 本文の解析を完全に回避する。添付率が低くフォロワー価値を損なうと分かった場合は、
   Phase 2 の text-rule で扱う。

決算短信サマリー内の予想欄が前回開示から変わっていても、短信からは本種別の候補を作らない。
予想の変更は修正開示そのものだけを事実として扱う（短信内の予想値は financial_fact として保存され、
Phase 2 以降の突合に使う）。

## 抽出（要確認）

修正開示の XBRL は、対象期間ごとに前回予想・今回予想・増減額・増減率・前期実績の行構造を持つ。
要素名・コンテキスト命名はフィクスチャで確定する。論理規則:

| 項目 | 規則 |
| --- | --- |
| forecast_period | 対象の期末年月と期間種別（通期 / 第 2 四半期累計）。複数期間の併記は期間ごとに別ファクト |
| metric | 決算実績と同じ正規化キーと要素候補リスト（net_sales / operating_profit / profit_owners） |
| previous_forecast | 前回発表予想に対応するコンテキストの値 |
| revised_forecast | 今回修正予想の値 |
| consolidation_scope | 連結を優先。連結がない会社のみ個別 |
| revision_delta | revised - previous。生値（円）で計算 |

- 指標・会計期間・連結範囲・単位が一致する previous と revised だけを組にする。
- 前回・今回の両方が数値として存在する metric だけを候補にする。
- previous_forecast が 0、または正負をまたぐ場合は増減率を出さない（保存のみ）。
- 開示に印字された増減額・増減率は financial_fact として保存し、自前計算と突合できる場合は
  検証に使う（矛盾したら抑止）。
- 修正理由は本文抽出が必要なため MVP では扱わない（→ [将来種別（凍結）](./future.md)）。

## 確定する企業ファクト

metric × forecast_period ごとに一つ。属性は決算実績と同形で、actual_raw / display_value の代わりに
previous_raw / revised_raw と previous_display / revised_display を持ち、baseline_type は使わない。

- previous_display / revised_display は決算実績と同じ規則で原文の表示文字列を保存する
  （bot は数値を再フォーマットしない）。
- display_unit は前回・今回で共通の一つとし、両者の scale・単位ヘッダが一致しない場合は
  unit_invalid で抑止する。

## 投稿テンプレート

~~~text
<企業名>（<コード>）｜業績予想
<対象期間>の<指標表示>予想を<修正前表示><単位>から<修正後表示><単位>に修正
~~~

表示規則（数値・期間・訂正表示）は決算実績と共通。「上方」「下方」「好材料」など、数値にない評価を
付けない。

## 優先度と重複キー

- priority = 100 + 指標順（営業利益 3、親会社株主に帰属する当期純利益 2、売上高 1）。
- 投稿重複キー:

~~~text
company_id / financial_forecast_revision / forecast_period（期末 + 期間種別） /
consolidation_scope / metric / previous_raw / revised_raw
~~~

前回・今回の値をキーに含める。再修正は値が変わるため新キーとなり投稿される。同じ修正を載せた
別文書は同キーで抑止される。訂正開示は correction_kind = source のバイパス（共通規則）。

## 投稿しない条件（抑止理由コード）

共通抑止に加えて:

| 理由コード | 条件 |
| --- | --- |
| no_xbrl | 表題は予想修正だが XBRL 添付がない |
| previous_undetermined | 前回予想が未定 |
| revised_undetermined | 今回予想が未定（予想の取り下げ） |
| range_forecast | レンジ予想（上限・下限の 2 値） |
| delta_mismatch | 開示印字の増減額と自前計算が矛盾 |
| pair_incomplete | 前回・今回のどちらかの値が欠ける |

未定・レンジの扱いを緩める判断は抑止統計を見てから行う（→ [決定ログ](../decisions.md)の未決事項）。

## フィクスチャ

| # | ケース | 確認すること |
| --- | --- | --- |
| 1 | 通期・連結の修正（3 指標） | 3 ファクト、本文が印字と一致、優先順 |
| 2 | 通期と第 2 四半期累計の併記 | 期間ごとの分割、文書上限 3 件の選抜 |
| 3 | 前回未定 | previous_undetermined 抑止 |
| 4 | レンジ予想 | range_forecast 抑止 |
| 5 | 配当予想の修正だけの開示 | 本種別の候補ゼロ（[配当](./dividend.md)側で処理、二重投稿なし） |
| 6 | 再修正 | 新キーで投稿される |
| 7 | XBRL なしの自由様式修正 | no_xbrl 抑止 |

受入基準は決算実績と同じ 4 条件。加えてフィクスチャ 5 の二重投稿なしを結合テストで確認する。
