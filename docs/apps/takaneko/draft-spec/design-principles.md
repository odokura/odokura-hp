# Takaneko 設計指針

## このページの役割

本ページは、Takaneko がどの価値を優先し、機能追加を何で判断するかを定める指針である。
種別ごとの抽出方法、投稿文、データ項目、抑止条件は[コンテンツ仕様](./content/common.md)に定める。

## プロダクトの約束

Takaneko は、日本株の一次情報を通じて、投資家が企業を自分の言葉で理解するための材料を積み上げる X bot である。

価値は開示を速く知らせること自体にはない。フォローを続けることで、事業の輪郭、経営の意図、実行の進み方、
財務・資本配分、リスクと制約について、判断の前提となる事実を少しずつ把握できることにある。

Takaneko は売買判断を代行しない。株価の速報や評価語で注意を引くのではなく、企業理解を前に進める一次情報の事実を、
X で読める大きさで継続して届ける。

## 価値の全体像

Takaneko は、文書の公開を通知するのではなく、投資家が次の問いに答えるための事実を届ける。

| 企業理解の問い | 提供するコンテンツ群 |
| --- | --- |
| この企業は何で収益を得ているか | 事業構造、製品・サービス、セグメント、地域、事業の開始・終了 |
| 経営は何を目指しているか | 戦略、数値目標、資源配分、投資・撤退方針 |
| 方針は実行されているか | 契約、提携、製品開始、設備、再編、進捗 |
| 数字と資金の使い方はどうか | 決算、業績予想、配当、自社株買い、資金調達、買収・売却 |
| 何が制約になり得るか | 下方修正、減損、訴訟、規制、供給・稼働上の制約 |
| 誰がどの体制で決めるか | 代表者、役員、組織、統治上の決定 |

各コンテンツ群は、さらに一つの主題を持つ企業ファクトへ分ける。一つの文書を要約して終わらせず、
同じ企業を継続して読むと事業・方針・実行・数字・リスクを結び付けられることを狙う。
どの問いから実装するかは[スコープと段階計画](./scope.md)で決める。

## 価値判断の枠組み

新しい機能、情報源、コンテンツは、次の価値を高める場合だけ採用する。

| 判断軸 | 提供すべき状態 |
| --- | --- |
| 企業理解 | フォロワーが、企業の事業・方針・実行・財務・資本配分・リスクを結び付けて考えられる |
| 具体性 | 文書の存在ではなく、企業について何が示されたかを一つの事実として理解できる |
| 中立性 | 一次情報の事実、定義済みの計算結果、会社の説明、bot の判断を混同しない |
| 継続性 | 一度の長い要約ではなく、独立して読める情報が積み上がる |
| 信頼性 | 投稿を内部の根拠、設定、計算、時点までたどり直せる |
| 読みやすさ | 深い理解に必要な情報を分割しても、重複や過剰投稿でタイムラインを損なわない |

## 設計上の原則

### 価値を先に定義する

新しいコンテンツは、どの企業理解の問いに答えるかを最初に定める。
実装可能でも、文書通知、数値の羅列、根拠のない評価にとどまるものは採用しない。

### 事実を最小単位にする

一投稿は、企業理解を一歩進める一つの事実である。複数の出来事を一つの短文に圧縮してはならない。
一つの文書に複数の価値があるときは、読める単位へ分割する。

### 自動化の範囲を限定する

自動投稿は、一次情報の根拠を持ち、定義済みの検証と表現で生成できるものだけに限る。
根拠があいまいな要約、将来への影響、良し悪しの評価は、自動化の対象にしない。

### 価値と安全を両立させる

手動承認なしの本番投稿では、根拠不足、鮮度切れ、重複、頻度超過、障害時の候補を投稿しない。
誤った投稿を避けることは、価値を届けないことではなく、信頼して継続利用できることの前提である。

## コンテンツ仕様への要求

提供価値として有効化するコンテンツには、実装前に次を定義する。

1. どの企業理解の問いに答えるか。
2. 受け入れる一次情報の文書種別と根拠の形。
3. 抽出する項目、比較・正規化の規則、欠損時の扱い。
4. 一企業ファクトへ確定する条件と、投稿しない条件。
5. X 上で一投稿にする表現テンプレート、重複キー、優先度。
6. 固定データで抽出・抑止・投稿文を再現する検証例。

この定義がない情報源・分類・表現を、本番の自動投稿へ追加してはならない。
有効化の手続きは[スコープと段階計画](./scope.md)のゲート、検証の中身は[検証計画](./verification.md)に従う。

## 初期の境界

- 対象は日本の取引所に上場する株式であり、外国株、為替、暗号資産、投資信託、利用者の口座・取引は扱わない。
- 上場企業の公式サイトと公的サイトで公開される一次情報を扱う。採用時に保存・手動クラウド退避・X 配信の可否を確認し、bot は取込時にライセンス判定をしない。
- リアルタイム株価、出来高、指数などの市場データは扱わない。
- 官庁・工業会の統計データは、X へ投稿する bot では扱わない。非公開のローカル分析サイトだけが
  セクター分析のために扱う（→ [ローカル分析サイト](./local-site.md)）。
- 正本は自宅の単一 PC 上の SQLite に無期限保存する。bot はバックアップ機能を持たず、運用者が定期的に手動で退避する。
- 高可用性や別環境への自動フェイルオーバーは初期要件に含めない。
