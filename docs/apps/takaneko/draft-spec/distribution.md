# ローカル表示

## 方針

Takaneko は作者本人だけが自宅 PC で使う private なアプリケーションである。企業ファクトは SQLite に保存し、
localhost の lab または CLI で確認する。X、メール、Slack、Web 公開、第三者への再配信は行わない。

本ページは旧「配信」仕様を置き換える。`post_candidate` と `post_attempt` は既存 SQLite の互換性のため
DDL に残るが、private 運用では新規の送信候補・送信試行を作らない。

## ローカル表示候補

company_fact は、次の条件をすべて満たす場合だけ作者本人の表示候補にする。

1. 対象銘柄集合と有効な ContentSpec に一致する。
2. 根拠箇所、必須属性、品質状態を検証できる。
3. 全体・情報源・ContentSpec のキルスイッチが解除されている。
4. 文書の訂正関係を確認できる。

表示候補には、企業名・コード・種別・対象時点・数値と単位・根拠箇所を併記する。固定テンプレートは
作者本人のローカル表示だけに使い、外部サービスの文字数制限や投稿頻度には従わない。

## 重複と訂正

同じ文書・根拠箇所・主題・ContentSpec 版から同じ company_fact を重複作成しない。訂正原本は既存記録を
上書きせず、新しい文書・根拠箇所・企業ファクトとして保存し、lab では訂正関係を表示する。

## スコープ外

- X API、OAuth、投稿、投稿結果照合、投稿の再試行
- 外部通知、外部 Web 公開、第三者へのデータ再配信
- 投稿頻度・文字数・タイムラインを前提とした選抜

外部配信を将来検討する場合は、情報源のライセンス、配信先の利用条件、本人による明示的な方針変更を
先に満たす別プロジェクトとして扱う。
