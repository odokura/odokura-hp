# ローカル分析サイト

作者本人だけが使う localhost 限定のローカルサイト。bot の収集結果・抽出・抑止・投稿候補・投稿記録を
サマリー表示し、官庁・工業会の統計データを使ったセクター動向の分析を行う。bot とはプロセス・
書き込み権・障害ドメインを分けて運用する。

以下、このコンポーネントを lab と呼ぶ。bot（収集・分析側）の仕様は他ページのとおりで、
本ページの機能は bot のマイルストーン・ゲートに影響しない。

## 位置づけと分離の契約

| 項目 | bot | lab |
| --- | --- | --- |
| 役割 | 開示の収集・正規化・企業ファクトの保存 | 表示・分析。外部へ配信しない |
| 公開範囲 | 作者本人の自宅 PC のみ | localhost のみ。外部公開しない（認証も持たない） |
| 実行 | 常駐（信頼性優先） | 任意起動。停止していても bot に影響しない |
| bot DB（SQLite） | 唯一の書き手 | 読み取り専用で参照（WAL 前提） |
| 統計データ | 扱わない | 独自の stats DB に保存（lab の正本） |
| 言語 | TypeScript（→ [アーキテクチャ](./architecture.md)の言語方針） | Python 第一候補（pandas・XBRL ライブラリが活きる） |

分離の規則:

- lab は bot の DB へ一切書き込まない。書き込みが必要な機能（設定変更・キルスイッチ）は lab に持たせず、
  bot の CLI で行う（→ [運用と設定](./operations.md)）。
- lab の読み取りは短いクエリに限る。長時間の読み取りトランザクションは WAL チェックポイントを妨げ、
  bot 側の DB 肥大を招くため禁止する。
- lab の障害・負荷・依存更新が bot の収集・保存へ波及しない構成を保つ。同一 PC で動かしてよいが、
  プロセス・依存関係・ログは分ける。
- lab は外部投稿・通知を生成しない。セクター分析は作者本人の localhost 閲覧だけに使う。

## 表示する内容（bot データのサマリー）

bot DB の読み取りだけで作れる画面。L0 の範囲。

| 画面 | 内容 |
| --- | --- |
| 収集状況 | 情報源ごとの最終取得時刻、取得・保存失敗、取得位置、欠損範囲 |
| 抽出・抑止統計 | ContentSpec・版ごとの候補数・確定数・抑止数、抑止理由コード別の件数推移（analysis_suppression と post_candidate から集計） |
| 投稿候補 | company_fact・post_candidate の一覧と、根拠（文書・根拠箇所・財務値）への突合表示 |
| 投稿記録 | post_attempt の一覧（dry_run・live、published・failed・unknown の別） |
| 銘柄ビュー | 銘柄ごとの企業ファクト時系列（決算・予想修正・配当の並び） |

投稿候補の画面は、[検証計画](./verification.md)の dry_run ゲート確認を効率化する
最初のユースケースである。ただし確認自体はログとリプレイ CLI だけでも可能とし、
lab の完成を bot のゲート条件にしない。

## セクター動向の分析

### 目的

複数の企業からなるセクターについて、所属銘柄の開示ファクト（bot が蓄積）と、官庁・工業会が公表する
統計データ（lab が収集）を並置し、セクターの環境変化と個社の数字を結び付けて読めるようにする。

### セクター定義

- セクターは手動定義の銘柄集合とし、lab の設定として版管理する。東証 33 業種など公的分類を
  参考にしてよいが、対象銘柄集合（30 銘柄以下）を運用者の関心で束ねる粒度を優先する。
- 一銘柄が複数セクターに属してよい。bot の対象銘柄集合とは独立に管理する（bot 設定を変更しない）。

### 統計データの収集と保存

- 対象は官庁統計（例: e-Stat の API で取れる鉱工業指数・機械受注などの月次系列）と、
  工業会の公表統計（例: 業界団体の月次受注・出荷統計）。具体的な系列の採用は運用者が決める。
- 採用前に、利用条件（プログラム取得の可否、保存の可否、出典明示の条件）を人手で確認する。
  ローカル表示のみで再配信しないため、bot の情報源より確認項目は少ない。X へ配信しない前提を崩す場合は
  再確認する。
- 系列はメタデータ（出典、名称、単位、頻度、取得 URL）と観測値（対象期間、値、取得時点）で保存する。
  統計は遡及改定されるため、取得時点の値を上書きせず追記し、最新取得分を表示既定とする。
- stats DB は bot DB と別の SQLite ファイルとし、lab が管理する。退避は運用者の手動で bot DB と
  同様に行う（頻度は低くてよい。統計は再取得で概ね復元できる）。

### 分析と表示

- 表示は、セクター所属銘柄の企業ファクト時系列と統計系列の並置、および定義済みの計算
  （前年同月比、移動平均など）に限る。計算定義と出典を画面に明示する。
- 将来予測・売買示唆・スコアリングのような bot の中立性契約と紛らわしい出力は、非公開であっても
  既定では作らない（作る場合も X の投稿系統とは明確に分離された lab 内に留める）。

## 技術方向

- Python を第一候補とする。統計処理（pandas）と、必要になった場合の XBRL 深掘り（Arelle 等の
  成熟ライブラリ）が理由。bot 側の言語判断は[アーキテクチャ](./architecture.md)を正本とする。
- L0 は Python 標準ライブラリの軽量 HTTP サーバーで実装する。追加依存や Web フレームワークは導入しない。
  `127.0.0.1` 固定で bind し、bot DB は SQLite URI の read-only 接続だけで開く。
- リポジトリは bot と同一に置き、ディレクトリ・依存管理（npm と Python 環境）・ログを分ける。

## 段階

| 段階 | 範囲 | 備考 |
| --- | --- | --- |
| L0 | bot データのサマリー表示（収集状況・抑止統計・投稿候補・投稿記録・銘柄ビュー） | bot のマイルストーンを待たずいつでも着手可。dry_run 運転の確認作業を効率化 |
| L1 | セクター定義、統計データの取り込みと保存、セクター画面 | 統計情報源の利用条件確認が前提 |
| L2 以降 | 系列の拡充、計算の追加 | 外部配信は lab の範囲外（X 投稿は bot だけが行う） |

bot の Phase（→ [スコープと段階計画](./scope.md)）とは独立に進める。
