# NeteClay 要件メモ

## 機能要件

| ID | 要件 | 優先度 |
| --- | --- | --- |
| F-01 | 起動後ワンタップで再生を開始できる（片手・抱っこ中を想定） | 高 |
| F-02 | 内蔵の既定値で、設定なしにそのまま再生できる | 高 |
| F-03 | 時間経過でテンポが変わる再生エンジン（速め → ゆっくり） | 高 |
| F-04 | 開始時に音量をフェードインする（いきなり大きな音を出さない） | 中 |
| F-05 | 入眠後、眠りが深くなるまで低音量で支え、その後フェードして自動停止する | 高 |
| F-06 | 固定フェードアウトで総再生時間の経過後に自動停止する | 中 |
| F-07 | 入眠までの時間・眠りが深くなるまでの時間をグラフィカルに調整できる（任意） | 中 |
| F-08 | 自動モード/手動モードを持ち、手動モードでは「寝た」で深まりフェーズへ移行できる。「もう一度あやす」は両モードで必ず使える | 高 |
| F-09 | セッションの操作イベントを記録する | 高 |
| F-10 | 記録から「だいたいの寝つき時間」を表示する | 中 |
| F-11 | メロディ系・環境音系の音色プリセットを持つ | 中 |
| F-12 | 音源パックを購入・復元できる（買い切り） | 中 |
| F-13 | バックグラウンド・画面ロック中も再生を継続する。通知権限がある場合、停止と「もう一度あやす」をロック画面/通知から操作できる。OS 制約で直接実行できない場合は通知タップでアプリへ戻し、即操作できる状態にする | 高 |
| F-14 | 最小限のオンボーディング（価値説明のみ・選択を求めない） | 中 |
| F-15 | 自動/手動モード、あやし直し、停止、中断復帰の状態遷移を一貫して扱う | 高 |
| F-16 | 初回再生失敗時に、寝かしつけを邪魔しない短いエラー表示と再試行導線を出す | 高 |

## 非機能要件

| ID | 要件 | 優先度 |
| --- | --- | --- |
| NF-01 | サーバーを使用せず、全機能が端末内で完結する | 高 |
| NF-02 | 購入・復元を除く寝かしつけ基本機能は完全オフラインで動作する（機内モードでも可） | 高 |
| NF-03 | 就寝フロー中の画面は暗色・低刺激・最小限にする | 高 |
| NF-04 | 就寝中に通知・ダイアログを出さない | 高 |
| NF-05 | バックグラウンド音声再生を継続し、着信等の中断から自動復帰を試みる。復帰できない場合は安全にセッションを閉じる | 高 |
| NF-06 | 音の途切れ・クロスフェードの破綻を起こさない | 高 |
| NF-07 | 連続動作のバッテリー消費を許容範囲に収める | 中 |
| NF-08 | 就寝フロー内に広告・購入訴求などの商業要素を出さない | 高 |
| NF-09 | 寝かしつけ中は他アプリ通知による覚醒を避けるため、OS のおやすみモード/集中モード利用を促す。ただしアプリ単体で他アプリ通知を強制的に遮断しない | 高 |

## 権限

- マイク・位置情報・センサーは使用しない。
- Android 13 以降は、再生中のメディア通知のために通知権限（`POST_NOTIFICATIONS`）を要求する。
  拒否されても再生自体は継続する。
- 通知権限を拒否された場合は再生継続のみ保証し、ロック画面/通知からの操作は非対応とする。
  操作はアプリ内で行う。
- 初回起動直後には通知権限を求めない。初回再生開始時に、ロック画面から停止するための権限として短く説明する。
- 他アプリ通知を一時的に抑えるには、OS のおやすみモード/集中モードを使う。アプリは設定案内を出せるが、
  ユーザーの許可なく OS 全体の通知を遮断しない。

## MVP 受け入れ条件

- 初回起動後、追加設定なしでホームの 1 タップから再生を開始できる。
- 無料のメロディ系 1 種と環境音系 1 種を選択できる。
- 自動モードで、入眠までの時間、深まり時間、固定フェードアウトに従って自動停止する。
- 手動モードで、「寝た」から深まりへ移り、「寝たを取り消す」で戻れる。
- 「もう一度あやす」を自動/手動の両モードでアプリ内から必ず実行できる。
- 手動モードで「寝た」を押し忘れても、90 分で安全停止する。
- 画面ロック・バックグラウンド中も、最大 90 分の検証再生で音が途切れない。
- 初回再生に失敗した場合、再試行でき、寝かしつけフロー内に長い説明や全画面ダイアログを出さない。
- 音源パック購入・復元で ownedPackIds が更新され、復元失敗時に既存の購入済み状態を消さない。
- 機内モードでも、購入・復元を除く寝かしつけ再生、設定、記録表示が動作する。
- 就寝フロー内に広告・購入訴求・音源パック案内を出さない。

## 対応言語

- 初版は日本語のみ。英語・繁体字は後続対応とする。

## リスク・確認事項

検証の進め方は[技術検証計画](./tech-verification.md)にまとめる。

- Expo / React Native でバックグラウンド音声再生を継続できるか。
- 2 トラック同時再生・長いクロスフェード・ギャップレスループの品質。
- 音源自作の制作スキル・制作時間がボトルネックになりうる（制作仕様は[音楽エンジン](./audio-engine.md)）。
- グラフィカル入力の初期値・範囲、および「眠りが深くなるまでの時間」の伝え方が実際の寝かしつけに噛み合うか。
