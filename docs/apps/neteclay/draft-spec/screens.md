# NeteClay 画面構成

## 方針

- 就寝前後に触る画面は暗色・低刺激・大きなタッチターゲットで統一する。
  抱っこしながら片手・薄暗い部屋での操作を前提にする。
- タブは使わず、ホーム中心のスタック構成。ルーティングは expo-router。
- ホームからの導線は「再生」と「設定」の 2 つだけ。こだわり設定は設定内の任意項目に置く。

## 画面遷移

```text
オンボーディング（初回のみ・価値説明のみ・スキップ可）
  └→ ホーム
       ├→ 再生中（フルスクリーン、停止はその場の確認 UI または長押し）
       │    └→（フェード完了 or 手動停止）→ ホーム
       └→ 設定
            ├→ こだわり設定（モードとタイムライン調整・任意）
            └→ 音源パック案内 / 購入 / 復元

ロック画面・通知の再生コントロール（バックグラウンド再生に付随）
  └→ 停止 / もう一度あやす（直接実行できない OS ではアプリ復帰）
```

再生中の状態遷移は[再生フロー](./playback-flow.md)を正とする。

## 各画面

### ホーム

- 画面の大半を占める再生ボタン 1 つ。タップで即再生開始。抱っこ中の誤タップに配慮し中央に大きく置く。
- 総再生時間の目安を小さく表示。
- 記録がたまっていれば「この子はだいたい○分で寝つく」を一言だけ小さく表示（グラフ・履歴一覧は置かない）。
  この表示は設定で非表示にできる。既定は控えめに表示する。
- 設定への導線はアイコン程度。購入訴求のバッジ・強調はしない。

### 初回起動・初回再生

- 初回起動では価値説明だけを見せ、モード選択・月齢入力・音量調整・プリセット選択を求めない。
- 初回プリセットは無料のメロディ系 1 種とする。
- 通知権限は初回起動直後には求めない。初回再生開始時に、ロック画面・通知から停止するために必要であることを短く説明してから求める。
- 通知権限を拒否されても再生は開始する。アプリ内操作だけで停止・あやし直しできるようにする。
- 初回再生で音声初期化に失敗した場合は、就寝フローを邪魔しない短いエラー表示にし、「もう一度試す」を出す。
  全画面ダイアログは出さない。再試行しても失敗する場合だけ、設定内のヘルプへ誘導する。
- 初回再生で選択中プリセットの音源ロードに失敗した場合は、settings.presetId を書き換えず、そのセッションだけ既定の無料プリセットで再試行する。
  既定無料プリセットも失敗した場合は、短いエラー表示で停止する。
- 音が鳴り始める前の初期化失敗は、寝かしつけセッションとして集計しない。
  Session を作成済みの場合は stopReason=error で閉じ、ホームの「だいたい○分」や直近 100 件の表示から除外する。
- 初回再生で音が出ない場合の詳細導線は設定内に置く。寝かしつけ画面の中ではトラブルシュートを長く見せない。
- 他アプリ通知を避けたい場合は、初回再生前の短い説明で OS のおやすみモード/集中モードを使うよう 1 回だけ案内する。
  アプリが勝手に有効化しない。設定内では同じ案内をいつでも見られるようにする。

### 再生中

- リアルタイム操作は任意。自動モードでは何も押さなくても既定スケジュールで完結する。
- 手動モードでは大きなボタンを 2 つ置く。
  - **「寝た」**（主操作・中央に最大）: 実際に寝ついたタイミングを合図する。1 タップで記録し、深まりフェーズへ移る。取り消しは cancel_asleep として記録し、深まり移行を取り消す。
  - **「もう一度あやす」**（副操作）: あやし直しに戻す。1 タップで反応する。
- 自動モードでは「寝た」をフェーズ制御として表示しない。画面上の主な任意操作は「もう一度あやす」と停止に絞る。
- 停止ボタンを置く。通常の手動停止は押下後にその場へ短時間だけ表示する停止確定 UI、または長押しで確定する。
  全画面ダイアログは出さない。通常停止は数秒の短いフェードで消音してから止め、緊急の即停止は長押しに割り当てる。
- 再生中のテンポ微調整は持たない。
- 暗色・低コントラスト・暖色寄り。白い閃光や輝度上昇はしない。
- 触覚フィードバックは既定オフまたは最小にする。端末が子どもに近い前提のため、必要な場合だけ設定でオンにできる。
- 開始数秒後に自動で減光し、ボタンの輪郭だけ残す。タップで一時的に UI を戻す。
- 任意で「画面を伏せる/さらに暗くする」操作を持つ。画面をほぼ消灯状態にし、タップで元の再生中 UI に戻す。
- 数値・時計の常時表示はしない（残り時間はタップ時のみ薄く表示）。
- タップ操作はセッションに記録される（[データモデル](./data-model.md)）。

#### 自動モードの再生中 UI

- 主表示は「流れている」ことだけが分かる静かな表示にする。
- 中央の大ボタンは置かない。親に寝たかどうかの判断を要求しない。
- 画面下部に「もう一度あやす」と停止を置く。
- 「もう一度あやす」は 1 タップで反応する。押したあとだけ短く「もう一度あやしています」と表示する。
- タップで UI を起こしたときだけ、薄く「見守り中」「もうすぐ静かに消えます」程度の状態文を表示する。

#### 手動モードの再生中 UI

- 中央に「寝た」を大きく置く。片手で押せるよう、画面下寄りではなく親指が届きやすい位置に置く。
- 「寝た」後は中央表示を「寝たを取り消す」に変え、一定時間だけ目立たせる。その後は小さな取り消し操作にする。
- 「寝た」取り消し後は、深まりフェーズを取り消して intro_hold に戻る。
- 「もう一度あやす」は常に残す。押したら「寝た」待ちに戻る。
- 手動モードでも残り時間の常時表示はしない。必要時だけ薄く表示する。

### 設定

- 音色プリセット選択（未購入プリセットには鍵表示、タップでパック案内へ）。
- こだわり設定（下記）への導線。
- ポリシー・規約リンク、購入の復元。
- おやすみモード/集中モードの案内。端末の通知設定へ誘導するだけで、アプリが OS 全体の通知を変更しない。
- ヘルプ、プライバシーポリシー、利用規約、購入復元、アプリバージョンを置く。
  ヘルプには「音が出ない」「通知/ロック画面操作」「おやすみモード/集中モード案内」を含める。

### こだわり設定（任意）

設定内の任意項目。既定パスの親は開かなくてよい（内蔵の既定値で成立する）。

- モード選択: 自動 / 手動。
  - 自動: 「入眠までの時間」に達したら自動で深まりフェーズへ進み、眠りが深くなるころにフェード停止する。
  - 手動: 「寝た」を押すまで深まりフェーズへ進まない。押した時点から眠りが深くなるまでの時間を数える。
- タイムライン調整: 横軸を時間、縦軸を音の落ち着きとして表示する小さなグラフで、次の 2 点だけを動かす。
  - 寝つくころ（入眠までの時間）。
  - 深くなるころ（眠りが深くなるまでの時間）。
- 総再生時間の目安を自動算出して表示（自動モードは必ずこの時間で自動停止する）。
- 開始テンポとフェードアウト時間は固定値とし、スライダーや編集項目として表示しない。
- 「既定に戻す」でモードと時間を内蔵の既定値に戻せる。
- 手動モードで記録された直近 100 件の「寝るまでの時間」を同じ画面の下部で見られるようにする。
  既定値を決めるための参考表示であり、自動提案や通知は行わない。

#### タイムライングラフ UI

- グラフはカード 1 枚に収める。カード内に曲線、2 つのハンドル、短い説明文を置く。
- ハンドル名は「寝つくころ」「深くなるころ」にする。
- グラフ下の説明は、選択中のモードに合わせる。
  - 自動: 「この時間で自動で見守ります」
  - 手動: 「寝たを押してから見守りに入ります」
- ハンドル操作中だけ分数を強調する。通常時は文章を優先する。
- ドラッグできない利用者向けに、選択中のハンドルを 1 分ずつ動かす補助ボタンを持つ。
- グラフの見た目は低刺激にする。鮮やかなアニメーションや点滅は使わない。
- グラフの値はドラッグ中に即時反映する。保存ボタンは置かない。
- 保存は UI 表示を即時反映し、永続化は短い debounce とドラッグ終了時の flush で行う。
  途中で画面を閉じても最後に見えていた値を残す。
- 再生中にこの画面で値を変えた場合、現在流れている音には反映しない。次回再生から反映する。
- ドラッグ中の値が制約に当たった場合は、その場でハンドルを止める。エラー文は出さない。

#### 手動モード記録

- タイムライン調整画面の下部に、手動モードで「寝た」を押した直近 100 件を新しい順に表示する。
- 1 行の表示は「日付」「寝るまでの時間」「もう一度あやす回数」程度に絞る。
- 「寝るまでの時間」は、最新の「もう一度あやす」から「寝た」までの時間とする。
  あやし直しがない回は、再生開始から「寝た」までの時間とする。
- 自動モードの推定値、cancel_asleep で取り消された回、sleepLatencyMs が null の回は一覧に出さない。
- 記録がない場合は「手動モードで寝たを押すと、ここに記録されます」程度の短い空状態を出す。
- 一覧は参考表示であり、タップして編集する詳細画面は MVP では持たない。

### プリセット選択

設定内の画面。寝かしつけフローの外でだけ触る。

| 状態 | 表示 | タップ時 |
| --- | --- | --- |
| 無料・未選択 | 名前、種別、短い説明、試聴 | 選択して settings.presetId を即時保存 |
| 無料・選択中 | 選択中表示、試聴 | 何もしない |
| 有料・購入済み | 名前、種別、パック名、試聴 | 選択して settings.presetId を即時保存 |
| 有料・未購入 | 鍵表示、パック名、低音量試聴 | パック案内へ遷移 |
| 音源ロード失敗 | 短いエラー、再試行 | 再試行。失敗が続く場合は既定無料プリセットを一時利用する |

- 試聴は寝かしつけフロー外でのみ、短尺・低音量で行う。
- 試聴は 15〜20 秒で自動停止する。別の試聴開始、画面離脱、本再生開始、購入画面遷移で必ず停止する。
  同時に鳴らせる試聴は 1 つだけにする。
- 試聴中に再生開始ボタンを押した場合は試聴を止め、寝かしつけ再生へ移る。
- 購入済みパックの復元後は、該当プリセットの鍵を外す。現在選択中のプリセットが有効になった場合も、自動で再生開始しない。
- 未購入プリセットを選んでも、寝かしつけフロー中に購入導線は出さない。
- 初期の無料プリセットは default_melody「やわらかメロディ」と default_ambient「しずかなノイズ」の 2 種。
  初回選択は default_melody とする。

### 音源パック案内・購入

- 買い切り。パックごとに、寝かしつけフロー外の設定画面でのみ短尺・低音量の試聴を付ける。
- 表示は未購入プリセット（鍵付き）選択時と設定内エントリのみ。寝かしつけフロー起動後は表示しない。

### オンボーディング

- 1 画面のみ。価値説明だけ:「抱っこのリズムに合わせて音楽が始まり、眠りに向かってゆっくりになる。
  寝ついたあとも低い音で見守り、深くなったころ静かに消える」。
- 選択を一切求めない。スキップして即ホームへ進める。

### ロック画面・通知コントロール

バックグラウンド再生に付随して OS が出すメディアコントロールを使う。アプリを開き直さずに操作できる。

- 停止。
- 「もう一度あやす」。
- 早送り/巻き戻し等の音楽的操作は出さない（誤操作で覚醒を招くため無効化）。
- ジャケット領域は暗い静止画にとどめ、光刺激を出さない。
- Android 13 以降で通知権限を拒否された場合は、再生継続のみ保証し、ロック画面/通知からの操作は非対応とする。
  操作はアプリ内で行う。
- 「もう一度あやす」は必須操作。OS のカスタムアクションで直接実行できる場合は直接実行し、
  できない場合は通知タップでアプリを開き、再生中画面の「もう一度あやす」にすぐ触れる状態にする。

| 条件 | 停止 | もう一度あやす |
| --- | --- | --- |
| 通知権限あり・カスタムアクション可 | ロック画面/通知から実行 | ロック画面/通知から直接実行 |
| 通知権限あり・カスタムアクション不可 | 通知タップでアプリ復帰後に実行 | 通知タップでアプリ復帰後、再生中画面ですぐ実行 |
| 通知権限なし | アプリ内で実行 | アプリ内で実行 |

## アクセシビリティ最低ライン

- 再生開始、寝た、もう一度あやす、停止、既定に戻す、プリセット選択は 44 pt 相当以上のタッチ領域を持つ。
- 主要ボタンにはスクリーンリーダー向けの短いラベルを付ける。
- タイムライングラフの 2 点は、ドラッグ以外に 1 分ずつ増減できる操作を持つ。
- Reduce Motion が有効な場合、グラフや再生中表示のアニメーションを最小にする。
