# NeteClay 設計判断と未決事項

確定した設計判断と、その理由を残す。仕様本体は各ページを正とし、ここは判断の要点をまとめる。

## 設計判断

| 判断 | 理由 |
| --- | --- |
| ターゲットは子どもの寝かしつけ。芯は「父親が楽をする」 | 疲れて子を抱いた親が半分眠りながら使えることを最優先にする |
| 既定パスはゼロ操作（内蔵の既定値 + ワンタップ + 放置で自動停止） | 選択・設定を求めない。タイムライン調整・タップは任意の上乗せ |
| 月齢などの入力を求めず、単一の既定値を土台にする | 月齢と寝つき時間の相関は個人差が大きく、選ばせる割に精度が伴わない。操作を減らす |
| タイミング推定にマイク・センサーを使わない | 技術ハードルと心理的負担が大きい。タイミングは既定値・タイムライン調整・再生中のタップで親が与える |
| サーバーを使わず全機能を端末内で完結する | プライバシーと運用コストの観点 |
| 音楽エンジンはテンポ段階別音源 + 長いクロスフェード | 実装が単純で音質を素材側で担保できる |
| テンポは自動では逆戻りしない（親の「もう一度あやす」でのみ戻す） | 自動で音量・テンポを上げ直すと子どもを起こすため |
| 眠りが深くなるまで低音量で支えるフェーズを設ける | 「置いたら起きる」対策。深くなるまでの時間を音でまたぐ |
| 開始テンポとフェードアウト時間は固定し、調整画面に出さない | 親が現場で判断しづらい音楽制御値を隠し、寝かしつけの体感に近い時間だけを扱う |
| 調整 UI はスライダーではなくタイムライングラフにする | 「寝つくころ」「深くなるころ」を感覚的に動かせるようにし、数値調整感を減らす |
| 自動モードと手動モードを持つ | 放置で終わる既定パスと、親が「寝た」を合図したい夜の両方に対応する |
| 再生状態遷移を再生フローに集約する | 画面・音楽エンジン・記録が別々に判断すると挙動がぶれるため |
| 手動モードにも安全上限を持ち、safety_timeout として記録する | 「寝た」の押し忘れで長時間鳴り続けることを防ぎ、親の手動停止と区別する |
| 「もう一度あやす」は必須操作とする | 布団移動や再ぐずりに対応する寝かしつけ中核機能のため |
| ロック画面・通知から停止/あやし直しできる導線を持つ | 直接あやし直しできない OS では通知タップでアプリへ戻し、すぐ押せる状態にする |
| 外向け文言では医学的効果を主張しない | ストア審査とユーザー期待の観点で、寝かしつけ環境の支援に留める |
| 開始時に音量をフェードインする | いきなり大きな音でうとうとした子を起こさない |
| 環境音系プリセット（ホワイトノイズ・心音等）を持つ | 低月齢の寝かしつけで定番。メロディ系だけでは低月齢に弱い |
| リアルタイム操作は「寝た」「もう一度あやす」の 2 つに絞る | 現場で迷わせない。反応は緩やかなランプにする |
| 記録は「操作イベントの保存」と「だいたい○分の表示」まで | 初期値の自動提案は精度・UX を詰めてから後続とする |
| 自動モードは総再生時間で必ず停止し、手動モードだけ安全上限を持つ | 自動はゼロ操作で終わらせ、手動は押し忘れ対策を別途持つ |
| 課金は音源パックの買い切り（サブスクなし）。寝つき機能は無料 | サーバーレスと相性がよい。眠りに直結する機能で差を付けない |
| 音源は自作（DAW） | ライセンス問題がなく、フェーズ構成・テンポ連動を最初から設計できる |
| iOS / Android の両 OS 対応 | Expo で両対応。検証は Android 先行、iOS を追試 |

## 未決事項

| 論点 | 次に決めること |
| --- | --- |
| タイムライングラフの操作感 | 2 点操作が直感的に理解できるかを試作で確認する |
| 「眠りが深くなるまでの時間」の既定値 | 実際の寝かしつけ時間の傾向・実利用のフィードバックで見直す |
| 有料パックの配布方式 | 音源容量を測定して、アプリ同梱 / App Bundle / On-Demand Resources から決める |
| expo-audio で足りるか | 技術検証 V-1・V-2 の結果で決める |
| ループ音源の書き出しフォーマット | V-2 で WAV と gapless AAC を比較 |
| 履歴・傾向の重い閲覧を将来 Premium にするか | 需要を見てから |
| アプリ名の正式表記 | ストア名・商標の確認 |
| 多言語対応の範囲 | 仮決定: 初版は日本語のみ。英語・繁体字は後続対応とする |
