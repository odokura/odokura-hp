# NeteClay 審査・スクリーンショット・プライバシー

ストア審査に添える前提、スクリーンショットの文言、データ安全性の申告をまとめる。
表現は[概要・安全ライン](./index.md)に従い、医学的効果を主張しない。

## スクリーンショット文言の例

| 画面 | 方向性 |
| --- | --- |
| ホーム | ワンタップで寝かしつけの音を開始 |
| 再生中（自動） | 抱っこ・トントンに合わせて、音がゆっくり落ち着く |
| 再生中（手動） | 「寝た」で深まりへ、「もう一度あやす」で戻す |
| こだわり設定 | 寝つくころ、深くなるころを感覚的に調整 |
| 音源パック | 好みに合わせて音色を追加（買い切り） |

- スクリーンショットは暗色・低刺激の実画面に近い見た目にする。白い閃光や強い輝度を避ける。
- 数値の睡眠測定・グラフ風の「睡眠スコア」的な演出は使わない（測定していないため）。

## 審査メモ（Review Notes）に入れる前提

- 医療アプリではない。睡眠測定・睡眠診断は行わない。
- マイク・センサー・位置情報・サーバーを使わない。すべて端末内で完結する。
- バックグラウンド音声再生を使う。就寝中の再生継続と、ロック画面/通知からの停止のために必要。
- 通知権限は初回起動直後には求めず、初回再生開始時に停止導線のためだと説明してから求める。拒否されても再生は継続する。
- 有料機能は音源パックの買い切りのみ。寝かしつけ再生の基本機能は無料。サブスクリプションはない。
- アカウント登録・ログインはない。オフラインで動作する（購入・復元を除く）。

## データ安全性の申告

iOS の App Privacy と Google Play のデータ安全性は、いずれも「データを収集しない」で申告する。
サーバーを持たず端末内に閉じるという[要件](../requirements.md)・[データモデル](../data-model.md)の方針に沿う。

| 申告項目 | 内容 |
| --- | --- |
| データ収集 | なし（端末内にのみ保存） |
| データ共有（第三者提供） | なし |
| マイク・カメラ・位置情報 | 使用しない |
| 連絡先・識別子・利用状況 | 収集しない |
| 暗号化・削除リクエスト | 端末内保存のみ。アプリ削除で端末内データも消える |
| 購入情報 | ストア（Apple / Google）の課金基盤に準拠。アプリはレシートを端末内で扱う |

- iOS: App Privacy は「Data Not Collected」を選ぶ。
- Google Play: データ安全性フォームで「データを収集しません」「データを共有しません」を選ぶ。
- 課金はストアの標準課金基盤を用いるため、決済情報をアプリが直接扱わない旨を審査メモに補足する。

## 年齢・レーティング

- 主対象は保護者。子ども向けカテゴリ（Kids / ファミリー）としては申請しない。
- 収集データがないため、レーティングは最も低い区分を想定する。実際の区分は各ストアの質問票の回答で確定する。

## 権限の説明文（ストア/OS ダイアログ）

- 通知権限: 「ロック画面や通知から再生を止めたり、もう一度あやすために使います。許可しなくても再生は続けられます。」
- バックグラウンド音声: 就寝中も音を続けるために使う旨を、必要なら審査メモで補足する。
