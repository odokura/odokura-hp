# NeteClay ビジュアルデザイン

画面の配色・タイポグラフィ・モーションの正をこのページに置く。
[画面構成](./screens.md)が「何を置くか」、このページが「どう見せるか」を決める。

## デザインコンセプト: アプリも一緒に眠る

NeteClay の画面は、子どもと一緒に眠りに落ちていく。
再生フェーズ（導入 → 深まり → 消灯）が進むにつれ、UI 自体の動きが遅くなり、色が沈み、最後は静止する。
このコンセプトから 3 つの原則を導く。

1. **時間への従属** — すべての動き・色の変化は再生フェーズに従属する。音楽と無関係に動く装飾は持たない。
2. **光の予算** — 暗い寝室で天井や親の顔に反射しない。画面が発してよい光の総量に上限を置く。
3. **シグネチャは一つ** — 象徴表現の中心は「呼吸する波紋」とする。背景アートは波紋に従属する「夜空」レイヤーとして扱い、波紋・再生ボタン・フェーズ/テンポの時間軸から独立して主張しない。

## 構図

美しさは配色より先に構図で決まる。トークンを守っても、構図が文書型なら破綻する。

### 原則

- **画面は文書ではなく一枚の絵**。主役は 1 画面に 1 つだけ。他のすべての要素の位置は、主役との空間的関係で決める。上から順にブロックを積む文書型レイアウト（タイトル → カード → 説明 → ボタン）を作らない。
- **余白は 0 か、spacing スケールのどちらか**。理由のない余白を作らない。額縁の中に額縁（画面 padding の内側にカード、カードの内側に padding）を作らない。
- **夜をカードに入れない**。背景グラデーションは画面全体（ステータスバーの裏からタブバーの裏まで）に敷く。角丸カードの中にグラデーションを描くと、背景が 2 つ競合して「枠が浮く」。
- **アプリ名・キャッチコピーのヘッダーをどの画面にも置かない**。ユーザーは開いたアプリの名前を知っている。アプリ名はアイコンとストアの仕事。
- **主役の中心は画面高の約 45%**（光学的中央。数学的中央は低く見える）。
- **発光・波紋は主役と同心**にする。別々にセンタリングされた円の重なりで「光っているように見せる」ことをしない。光は主役から発する。

### ホームの構図

構成要素は 3 つだけ。**テキストは置かない**。

```text
┌───────────────────────┐
│（ステータスバー透過）      │
│                       │
│       ・  ・  ・        │
│     ・         ・      │   波紋は再生ボタンを中心に発生
│    ・   ╭───╮   ・     │
│    ・   │ ▶ │   ・     │ ← 琥珀の円。中心は画面高の約 45%
│    ・   ╰───╯   ・     │
│     ・         ・      │
│       ・  ・  ・        │
│                       │
│（グラデーションが継続）    │
│                       │
│      ⌂        ♪       │ ← タブバー。半透明でグラデーションが透ける
└───────────────────────┘
```

1. full-bleed のグラデーション背景（待機セット）
2. 呼吸する波紋（再生ボタンと同心）
3. 再生ボタン（薄く光る琥珀のリング）

再生中は同じ画面で 3 リングに展開する（画面遷移なし）。

```text
┌───────────────────────┐
│（ステータスバー透過）      │
│                       │
│       ・  ・  ・        │
│     ・         ・      │
│    ・  ○  ◉  ○  ・    │ ← ○=中立色リング  ◉=琥珀リング（寝た）
│    ・  ■  )))  ↺  ・   │   グリフ: ■ / 三日月 / 循環矢印
│     ・         ・      │   中心は画面高の約 45%
│                       │
│         ~~~~~         │ ← グラフ（再生中のみ表示。低刺激な曲線）
│                       │
│      ⌂        ♪       │ ← タブバー（再生中も表示したまま）
└───────────────────────┘
```

- モード名・総再生時間・寝つき目安など、状態や数値をホームに出さない。寝かしつけの開始に必要な情報ではない。
- 通知権限・おやすみモードの説明は、初回再生タップ直後のフロー内で出す（[画面構成](./screens.md)）。ホームに常駐させない。
- エラー時のみ、ボタンの下に 1 行のエラー文（danger）を出す。再試行はボタン自体が担う。
  エラーが続く場合のヘルプ誘導は、その 1 行の下に小さく置いてよい。
- タブバーの背景は半透明とし、グラデーションが透けて画面と連続させる。

### 他画面の構図

- 再生中もホーム画面のまま（画面遷移なし）。full-bleed 継続。主役は波紋、従は 3 リングとグラフ。テキストは状態文 1 つまで（エラー時のみ）。
- 選曲・設定は、title スタイル（20）の画面見出し 1 つまで許容する。アプリ名は置かない。
- 全画面共通で、コンテンツを角丸カードに包んで背景から切り離す構造を既定にしない。
  面の区切りが必要なときだけ bg.raised を使う（リスト行・グラフカードなど）。

## カラー

### 基調と役割

基調は**夜のインディゴ**（青みの近黒）、アクセントは**常夜灯の琥珀** 1 色のみ。
寒色の闇の中に琥珀の小さな灯りが 1 つある、という夜の子ども部屋の構図を画面に写す。

| トークン | 値 | 用途 |
| --- | --- | --- |
| bg.deepest | #070A14 | 消灯フェーズ・減光時の到達点 |
| bg.base | #0B0F1E | 画面の基本背景 |
| bg.raised | #151A2E | カード・リスト行 |
| bg.overlay | #1D2340 | 押下中の面・選択行 |
| line | #262C4A | 罫線・アウトラインボタンの枠 |
| text.primary | #E9EBF5 | 主要テキスト（純白は使わない） |
| text.secondary | #9AA2C0 | 補助テキスト・非活性アイコン |
| text.tertiary | #5C6483 | 装飾的テキスト。重要情報には使わない |
| accent.primary | #EFBD8D | 再生ボタン・現在位置・選択状態。琥珀 |
| accent.deep | #B98A5E | 減光時・非活性時の琥珀 |
| accent.onAccent | #221609 | 琥珀面の上に載せる文字・アイコン |
| danger | #D9928F | エラー文のみ。夜に鳴らさない弱い警告色 |

- アクセントに 2 色目を追加しない。琥珀は「画面内で最も大事な 1 つ」にだけ使う。
- 現行実装の灰緑系（#263230 ほか）は全廃する。
- 停止ボタンに danger を使わない。停止は破壊操作ではなく、中立色のアウトラインで表す。

### フェーズによる背景変化

背景グラデーションは再生フェーズで沈んでいく。切り替えではなく、音のランプと同じく数十秒かけて連続変化させる。

| 状態 | グラデーション（上 → 下） | 意図 |
| --- | --- | --- |
| 待機（ホーム） | #101736 → #0B0F1E → #070A14 | まだ起きている夜。再生ボタン背後にだけ琥珀の微光 |
| 導入 | #0E1430 → #090D1B | 音が流れ始め、少し沈む |
| 深まり | #0A0E22 → #070A14 | 眠りに寄り添う。もっとも暗い動的状態 |
| 消灯 | #070A14 単色へ収束 | 波紋も止まり、画面ごと眠る |

### プリセット背景アート

ホームの選曲背景は、プリセットごとのグラデーションと干渉リング模様で構成する。これは波紋に従属する「夜空」レイヤーであり、波紋・再生ボタンより前面に出さない。スワイプ時はプリセットページごとに背景アートが切り替わり、購入状態には依存しない。未知の presetId は default_melody の設定にフォールバックする。

| プリセット | 背景グラデーション（上 → 下） | 波源色 | 波源数 |
| --- | --- | --- | --- |
| default_melody（やわらかメロディ） | #181A38 → #0B0F1E → #070A14 | #EFBD8D（琥珀） | 2 |
| default_ambient（しずかなノイズ） | #131A2E → #0A101F → #070A14 | #9BB0D4（青灰） | 3 |
| calm_pack_1_lullaby（ゆりかごの余韻） | #1D1834 → #0D0E22 → #070A14 | #C4A8C9（ラベンダー） | 2 |
| calm_pack_1_ambient（夜明け前のしずけさ） | #102028 → #0A121E → #070A14 | #8FB4B8（ミュートティール） | 3 |

波源色はいずれも低彩度に抑え、純白や高彩度の寒色発光として扱わない。波源は同心円リング約 10 枚のスタックとし、fill は使わず、外側のリングほど薄く消えるようにする。

| フェーズ | 軌道速度 | 軌道半径 | 波源不透明度 | グラデーション |
| --- | --- | --- | --- | --- |
| idle | 1.0× | 1.0× | 1.0× | プリセット固有 |
| intro | 1.3× | 1.0× | 1.0× | phaseBackgrounds.intro へクロスフェード |
| deepening | 0.7× | 0.5×（中心へ収束） | 0.8× | phaseBackgrounds.deepening へ |
| fading / stopping | 0.05×（ほぼ静止） | 0.3× | 0.3× | phaseBackgrounds.fading へ |

波源は 2〜3 個置き、19 秒 / 31 秒 / 47 秒の互いに素な周期でゆっくり楕円軌道を漂わせる。同一プリセット内で周期を重複させない。波源の中心は画面中央付近を基準にしつつ、上寄り・下寄りへ分散させる（centerY 0.3〜0.7 程度）。円の重なりで干渉縞を生み、単調な回転リングにしない。

グラデーション切替はプリセットグラデーションの上にフェーズグラデーションを重ね、約 8 秒で opacity をクロスフェードする。フェーズ変更時の軌道速度・軌道半径・波源不透明度は約 4 秒で滑らかに遷移させ、瞬間切替を見せない。

再生中は、背景アート全体を波紋と同じ周期で 0.99〜1.01 の範囲で呼吸させる。周期は breathingCycleMsForPreset 由来とし、メロディ系では BPM、環境音系では固定周期に従う。Reduce Motion 有効時は波源を初期位置で静止させ、パルスを止め、グラデーション切替は即時にする。

### コントラスト規定

- text.primary / bg.base: 約 14:1。本文・ボタンラベルに使う。
- text.secondary / bg.base: 約 7:1。補助情報まで。
- text.tertiary / bg.base: 約 4:1。装飾のみ。この色で必須情報を出さない。
- accent.onAccent / accent.primary: 約 10:1。琥珀面の文字。
- 減光状態はコントラスト規定の対象外とする（タップで通常表示に戻せるため）。

## 光の規律（輝度予算）

「かっこいい発光」と「暗い部屋で眩しくない」を両立させるための上限。

- 純白 #FFFFFF と、彩度の高い寒色の発光は使わない。
- 再生中画面は、面積の 9 割以上を bg 系（明度の低い色）で構成する。
- グローは琥珀の透明度だけで作る（最大でも不透明度 12% 程度）。白いハロー・ぼかし光は使わない。
- 減光状態では全要素の不透明度を約 0.35 に落とし、琥珀は accent.deep に置き換える。
- ホームなど就寝フロー外の画面も同じパレットを使い、昼画面と夜画面で世界観を分けない。

## タイポグラフィ

- フォントはシステムフォント（ヒラギノ / Noto Sans JP）。カスタムフォントは導入しない。
- ウェイトは **400 と 600 の 2 つだけ**。700 は使わない（現行実装の全 700 を置き換える）。

| スタイル | サイズ / ウェイト | 用途 |
| --- | --- | --- |
| display | 34 / 600 | 分数など画面に 1 つの主数値。等幅数字（tabular-nums） |
| title | 20 / 600 | 画面見出し・カード見出し |
| body | 15 / 400・行送り 1.6 | 本文 |
| caption | 12 / 400 | 補足・ラベル |

- 分数・時刻は必ず等幅数字にする（桁が変わっても幅が揺れない）。
- **1 画面に置く本文の文は最大 2 つ**。それ以上の説明はオンボーディングとヘルプへ退避する。
  [画面構成](./screens.md)の「文字は極力減らす」方針の数値化。

## 形と奥行き

- 角丸は 12 / 20 / 28 / 完全円の 4 段のみ。中間値を作らない（現行の一律 8 を全廃）。
- **ドロップシャドウを使わない**。ダークテーマでは影が濁って見えるため、奥行きは bg.base → raised → overlay の明度差で表す。
- 面の境界が必要な場所は 1px の hairline（line 色）で示す。

## シグネチャモーション: 呼吸する波紋

再生ボタンの背後に同心の波紋を 3 枚置き、呼吸のリズムで明滅させる。静止画にしない。

### 周期の規定

| 状態 | 周期 | 根拠 |
| --- | --- | --- |
| 待機（ホーム） | 5.0 秒 | 大人の安静呼吸（約 12 回/分）。まだ音は鳴っていないので音楽とは独立 |
| 再生中（メロディ系） | 1 波 = 4 拍 | 楽曲テンポに同期する。75 BPM で 3.2 秒、50 BPM で 4.8 秒。テンポが落ちると波紋も遅くなる |
| 再生中（環境音系・bpm null） | 6.0 秒固定 | テンポ概念がないため、深い呼吸の速さに固定 |
| 消灯 | 振幅を 0 へ収束 | フェードアウトと同時に波紋も眠る |

- テンポは連続的に落ちるため、波紋の周期も同じランプで連続変化させる。段階的な切り替えを見せない。
- 「音と同じ時間軸で UI が遅くなっていく」ことがこのアプリの視覚的アイデンティティである。

### 描画パラメータ

- 3 枚のリングを位相 1/3 ずつずらし、scale 1.00 → 1.05、不透明度 0.28 → 0.50 を正弦イージングで往復する。
- リングの色は bg.overlay 系の明度違いとし、琥珀にしない（琥珀は再生ボタンだけ）。
- Reduce Motion 有効時は scale を固定し、不透明度のみ ±0.06 のゆらぎ、または完全静止にする。

### 操作フィードバック

- 押下は scale 0.97・120ms のみ。OS のリップルエフェクト・触覚は使わない（[決定ログ](./decisions.md)）。
- 実装は react-native-reanimated を用いる。新たな描画ライブラリ（Skia 等）は、この仕様で表現が不足すると確認できるまで導入しない。

## アイコン

- 当面は @expo/vector-icons の線画系で仮置きし、後日イラストへ差し替える（[画面構成](./screens.md)の方針）。
- 線の太さ 1.5〜2px 相当・丸端で統一する。塗りつぶしは「選択中」の表現にだけ使う。
- サイズは 24（タブは 26）。色は text.secondary、活性・選択中のみ accent.primary。
- イラスト差し替え時もこのサイズ・色規則を引き継ぐ。

## コンポーネント指針

### 再生ボタン（ホーム）

塗りつぶした円ではなく、**夜に浮かぶ薄く光るリング**にする。文字は置かない。

- ヒット領域は直径 120pt 程度の完全円。視覚は 3 層で構成する:
  1. リング: 線幅 2px 相当、accent.primary、不透明度 0.85 程度
  2. リングの外側ににじむ琥珀のグロー（不透明度 4〜6%。輝度予算内）
  3. 中央に再生の三角形グリフ（accent.primary）。「再生」等のテキストラベルは置かない
- 円の内側は塗らない。背景の夜がそのまま透け、リングだけが灯る。
- 波紋はこのリングと同心に呼吸する（リングの延長として波紋が広がって見えること）。
- 押下: リングの不透明度をわずかに上げ、scale 0.97 / 120ms。
- 準備中はリングをゆっくり明滅させる。「準備中」の文字を出さない。
- accessibilityLabel は「再生を始める」を維持する。
- 「常夜灯を灯す」操作として、画面の主役はこのリングと波紋だけにする。
- リングの色は琥珀とする。参照イメージにある青白い光は、禁止事項（高彩度の寒色発光・純白）に
  当たるため採らない。形（細い光の輪 + 中央グリフ + 微グロー）だけを採る。

### 再生中の 3 リング（ボタン展開）

再生ボタンと同じリング設計。塗り円にしない。

**展開アニメーション（再生開始時）**:
- 再生ボタンリングが scale 1.0 → 0.8 にしぼみながらフェードアウト（120ms）
- 3 リングが中央から同時に出現: 停止は左（translateX -96pt）、寝たは中央、寝ないは右（+96pt）に滑り出す（300ms / ease-out）
- グラフが下部にフェードイン（200ms 遅延後）

**収束アニメーション（停止時）**:
- 逆方向に 250ms / ease-in で収束し、再生ボタンリングに戻る

| ボタン | リング色 | グリフ | 補足 |
| --- | --- | --- | --- |
| 停止 | line (#262C4A) | ■（フィル正方形） | 中立色。danger にしない |
| 寝た | accent.primary (#EFBD8D) | 三日月グリフ | 主操作を琥珀で示す |
| 寝ない | line (#262C4A) | 循環矢印グリフ | 中立色 |

- 3 リングのサイズは直径 80pt（ヒット領域は 88pt 以上確保）。線幅 2px。内側は塗らない。
- 再生ボタン（120pt）と同じ glow 規則を継承する（outer glow 不透明度 4〜6%）。
- 琥珀を使うのは「寝た」リングとグラフの現在位置のみ。停止・寝ないは line 色で統一する。

### タブバー

- bg.base の半透明（不透明度 0.92）+ 上辺 hairline。ぼかし（blur）は使わない（Android の描画コスト回避）。
- アイコンのみでラベルは置かない。活性タブだけ accent.primary。

### タイムライングラフ（調整・再生中とも）

- 曲線は 2px、line 色より一段明るい程度に抑える。
- 現在位置は琥珀のドット（6pt）+ 微光。琥珀を使うのはこのドットとハンドルだけ。
- 軸線・目盛り数字は描かない。数字はハンドル操作中の吹き出しのみ（[パラメータ](./parameters.md)）。

## 画面別の適用メモ

| 画面 | 背景 | 琥珀の使いどころ |
| --- | --- | --- |
| ホーム | 待機グラデーション + 波紋 | 再生ボタンのみ |
| 再生中 | フェーズ連動グラデーション + 波紋 | 「寝た」ボタンとグラフの現在位置 |
| 選曲 | bg.base + raised カード | 選択中プリセットの枠と再生中マーク |
| 設定・こだわり設定 | bg.base | タイムラインのハンドルと現在値 |
| オンボーディング | 待機グラデーション | 進むボタンのみ |

## 禁止事項

- 純白 #FFFFFF、高彩度の寒色発光。
- ドロップシャドウ。
- アクセント 2 色目の追加。
- fontWeight 700。
- 現行の灰緑パレット（#263230 系）の残留。
- 赤系・danger 色の停止ボタン。
- OS 標準の青いリンク色（リンクも text.secondary + 下線で表す）。
- 音楽と無関係に動き続ける装飾アニメーション。ただし、背景アートの待機時ドリフト（19 / 31 / 47 秒周期）は、待機時の波紋（5 秒呼吸）と同じく、就寝前の夜空レイヤーとして許容する。

## 段階導入と検証

1. **トークン基盤**: theme.ts 新設と hex 直書きの全廃。ここだけで統一感の問題は解消する。
2. **シグネチャ**: 波紋の呼吸アニメーションと BPM 同期。reanimated 導入。
3. **コンポーネント**: ボタン・タブバー・アイコンの刷新。
4. **画面別仕上げ**: ホーム → 再生中 → タイムライン → 選曲 → 設定の順。
5. **暗所テスト**: 実際の寝室の照度で実機確認する。天井への反射・顔への映り込みがあれば輝度予算を締める。
