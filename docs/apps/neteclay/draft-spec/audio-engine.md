# NeteClay 音楽エンジン

テンポ・音色・音量は、時間と[パラメータ](./parameters.md)の関数として決まる。
親のリアルタイム操作があれば、それで位相を上書きする。

## 要求

- 開始時は抱っこ・トントンに合う比較的速めのテンポで始める。音量は数秒かけてフェードインし、
  いきなり大きな音を出さない。
- 「入眠までの時間」をかけてテンポを漸減する。
- 入眠後は「眠りが深くなるまでの時間」の間、低く安定した音で支える（生活音のマスク）。
- そのあとフェードアウトして自動停止する。
- テンポ・音色・音量の変化は聴感上「切り替わった」と気づかれないこと（不連続な変化は覚醒を招く）。
- 停止は 1 タップで即停止する。急停止のクリックノイズを避けるため最短のフェード（数百 ms 程度）だけかける。長押しや確認 UI は挟まない。

## フェーズ設計

開始テンポとフェードアウト時間は固定値、各フェーズの時刻は[パラメータ](./parameters.md)の値で決まる。

| フェーズ | 期間 | テンポ | 音色・音量の方向 |
| --- | --- | --- | --- |
| 導入 | 0 →「入眠までの時間」 | 固定開始テンポ（75）→ 約 55 へ漸減 | メロディあり → パッド寄りへ。徐々に静かに |
| 深まり | 入眠後 → +「眠りが深くなるまでの時間」 | 約 50 で一定 | 低音量の安定した音。リズム感を消す。生活音をマスク |
| 消灯 | 深まり終了後 | — | 固定フェードアウト時間（5 分）で音量ゼロまで下げ停止 |

- テンポは時間に対して基本的に単調減少する。自動では逆戻りしない。
- 深まりフェーズは「置いたら起きる」対策。眠りが深くなるまでの時間を低い音でまたぎ、
  物音や布団への移動での覚醒を防ぐ。
- 自動モードでは「入眠までの時間」に達したら深まりフェーズへ進む。
- 手動モードでは親が「寝た」を押すまで深まりフェーズへ進まず、低刺激な導入状態で待つ。

## リアルタイム操作による位相遷移

再生中、親のタップで実際のタイミングに合わせられる（任意。詳細は[パラメータ](./parameters.md)）。
状態遷移の正は[再生フロー](./playback-flow.md)とする。

- **「寝た」**: 自動・手動の両モードで押せる。現在地から「深まり」フェーズへ緩やかに移行し（トーンダウン）、fell_asleep を記録する。以降の深まり・消灯の長さは切り替え時点から測り直す。自動モードでは押さなくても目安時刻で深まりへ進む。
- **「寝ない」**: 「導入」寄りのテンポへ緩やかに戻す。親の明示操作のときだけ許可し、自動では戻さない。
- 自動モードで親がタップしなければ、既定のスケジュールがそのまま進む。
- 手動モードで親が「寝た」を押さなければ、深まり・消灯には進まず、導入の終端付近で低刺激に待つ。
- どの遷移も等パワークロスフェード + テンポのランプで行い、数十秒かけて寄せる（急な変化を作らない）。
- ロック画面・通知からの「寝ない」は、直接実行できる OS では直接実行する。直接実行できない場合は
  通知タップでアプリを開き、再生中画面の「寝ない」にすぐ触れる状態にする。停止は確実に動作させる。

## タイマー規則

- 再生開始時刻、現在フェーズの開始時刻、最後の「寝ない」時刻を保持し、画面復帰時は現在時刻との差分で状態を再計算する。
- 自動モードでは、start から入眠までの時間が経過したら deepening に入る。
- 手動モードでは、start から入眠までの時間が経過しても deepening には入らず、intro_hold として低刺激な導入状態を保つ。
- 手動モードで「寝た」を押した時点を deepening の開始時刻とする。
- deepening の開始から「眠りが深くなるまでの時間」が経過したら fading に入る。
- fading は固定 5 分。途中で「寝ない」が押されたら、短いフェードインを挟んで intro に戻す。
- 「寝ない」が押されたら、その時刻を新しい起点として入眠までの時間を数え直す。手動モードでは再び「寝た」待ちに戻る。
- fading 中に「寝ない」が押された場合は、停止予定を取り消し、pendingStopReason を破棄してから intro に戻す。
- 手動モードで「寝た」が押されない場合も、再生開始または最後の「寝ない」から 90 分で安全停止する。
  この停止の stopReason は safety_timeout とする。
- バックグラウンド中もタイマーは止めない。アプリ復帰時に、現在時刻から本来いるべきフェーズへ追従する。
- OS 中断から復帰できた場合は、中断前のフェーズと経過時刻をもとに再開する。復帰できない場合は interrupted として閉じる。
- 自動モードの最大再生時間は 30 分 + 40 分 + 5 分 = 75 分。手動モードの安全停止検証を含め、実装検証は 90 分で行う。
- 再生開始時点の params と presetId をそのセッションのスナップショットとして使う。再生中に設定が変わっても、現在の音には反映しない。

## 音量方針

- 端末音量が高くても、アプリ基準ラウドネス（目安 -20 LUFS）と開始フェードインにより急な大音量を出さない。
- 初回再生時に音量警告を出さず、安全に始められる音量設計にする。
- プリセット間はラウドネス正規化済みにし、切り替えても音量感が変わらないようにする。
- Bluetooth・外部スピーカー出力でも同じ基準で扱う。

## 実装方式

**テンポ段階別の事前レンダリング + 長いクロスフェード**を採る。各プリセットについてフェーズ分の
音源を用意し、30〜60 秒のクロスフェードで移行すれば、聴感上は連続的に感じられる。移行タイミングは
時間で決まるため、エンジンは「2 トラックのクロスフェーダー + タイマー」に単純化できる。

### 音声スタック検知

再生中にプレーヤーが停止状態に陥ることを検知するため、expo-audio のステータスコールバックを監視する。

- 再生開始後にプレーヤーが停止状態を 5 秒以上継続した場合を「スタック」と判定する。
- スタック検知時はセッションを `stopReason=error` で閉じ、エラーとして扱う。
- スタック検知の猶予（5 秒）は、短い一時停止（着信・割り込み）との誤検知を避けるためのものであり、復帰できた場合は通常の割り込み処理に委ねる。

リアルタイムのタイムストレッチやプロシージャル生成は、音質の担保と実装コストの面で採らない。

## 音色プリセット

- プリセット = 各フェーズ分の音源セット + メタデータ（固定開始テンポ、固定フェード時間）。
- 2 系統を持つ:
  - **メロディ系**: 「オルゴール」「アンビエントパッド」「雨とパッド」など。テンポ変化が効く。
  - **環境音系**: 「ホワイトノイズ（シューッ）」「心音・胎内音」など。低月齢の寝かしつけで定番。
    テンポの概念が薄いので、テンポ変化は音量・帯域の変化に読み替える。「深まり」フェーズと相性がよい。
- 各系統から少なくとも 1 種を無料で持つ。追加は音源パックとして販売（[マネタイズ](./monetization.md)）。
- 初回起動時の既定プリセットはメロディ系の 1 種とする。
- 全プリセットで音量ラウドネスを正規化し、切り替えても音量感が変わらないようにする。

### MVP 初期プリセット

| presetId | 表示名 | 種別 | 位置づけ |
| --- | --- | --- | --- |
| default_melody | やわらかメロディ | melody | 初回再生の既定。抱っこ・トントンのテンポ感を持つ |
| default_ambient | しずかなノイズ | ambient | 低月齢向け。テンポより音量・帯域変化で落ち着かせる |

- 上記 2 種は無料同梱とし、packId を持たない。
- 有料パックのプリセット ID と表示名は、音源制作完了後に音源パック仕様へ追記する。
- 選択中プリセットのロードに失敗した場合は 1 回だけ同じプリセットを再ロードし、それでも失敗したらそのセッションだけ default_melody を試す。
  ユーザーの presetId は自動で書き換えない。
- default_melody も失敗する場合は、短いエラー表示にして音声エンジンを停止し、セッションを error で閉じる。

検証用音源は次のコマンドで生成する。

```bash
npm run audio:verification
```

生成物は `assets/audio/presets/` 配下の 8 ファイルとする。WAV は git にコミットせず、生成スクリプトを正とする。

| default_melody | deepening | 50 | 10 小節 = 48.0 秒 | 2,116,800 | C2/G2/C3 のパッドのみ。プラックなし |
| default_ambient | intro | null | 15.0 秒 | 661,500 | ローパスしたノイズドローン。やや動きあり |
| default_ambient | deepening | null | 15.0 秒 | 661,500 | ローパスしたノイズドローン。帯域を絞る |
| calm_pack_1_lullaby | intro | 72 | 3 小節 = 10.0 秒 | 441,000 | default_melody と同ロジックの低めの変形 |
| calm_pack_1_lullaby | deepening | 48 | 3 小節 = 15.0 秒 | 661,500 | 低めのパッドのみ |
| calm_pack_1_ambient | intro | null | 15.0 秒 | 661,500 | ローパスしたノイズドローン。音色違い |

- フォーマットは 44100Hz / 16bit PCM / モノラル WAV とする。
- 各ファイルは RMS -20dBFS を目安に正規化し、ピークが -3dBFS を超えないようにする。
- 末尾をまたぐノートのリリースはバッファ先頭へ折り返して合成し、ループ継ぎ目のクリックを構成的に避ける。
- LFO・ゆらぎの周期はループ長の整数分の一とし、継ぎ目で位相が合うようにする。

## 音源アセット仕様

MVP では、1 プリセットを「導入」「深まり」の 2 種類のループ音源で構成する。消灯は専用音源を持たず、
深まり音源を固定 5 分でフェードアウトする。

| アセット | 用途 | 目安 BPM / 性質 | ループ長 | 必須 |
| --- | --- | --- | --- | --- |
| intro | 導入。抱っこ・トントンに合う出だし | 75 BPM 相当 | 2〜4 分 | 必須 |
| deepening | 深まり。低音量で見守る | 50 BPM 相当、またはテンポ感の薄い環境音 | 2〜4 分 | 必須 |
| artwork | ロック画面・通知用の暗い静止画 | 暗色・低刺激 | 1 枚 | 任意 |

- メロディ系は intro と deepening で音数・高域・リズム感を明確に落とす。
- 環境音系は BPM を厳密に持たなくてよい。intro はやや動きのある音、deepening は帯域を落とした安定音として扱う。
- ファイル名はプリセット ID、フェーズ、フォーマットが分かる形にする。

```text
presets/{presetId}/intro.wav
presets/{presetId}/deepening.wav
presets/{presetId}/artwork.png
```

- MVP の無料同梱はメロディ系 1 種、環境音系 1 種を目安にする。
- 有料パックは 1 パックあたり 2〜4 プリセットを目安にする。
- 同梱サイズが過大になる場合は、無料同梱を優先し、有料パックの配布方式を[技術検証](./tech-verification.md)で決める。

## 音源の制作仕様（自作・DAW）

- 各フェーズ 2〜4 分のシームレスループ（ループポイントで無音・クリックが出ないこと）。
- フェーズ間はクロスフェード前提のため、同一キー・近い音量感で制作する。
- BPM をファイル名とメタデータに明記する（例: 導入 75 / 深まり 50）。
- ラウドネスは全ファイルで統一（目安 -20 LUFS 程度の小音量基準。寝かしつけ用のため低めに）。
- 高域は深まりフェーズに向けて段階的に減らす（ローパスをかけた状態で書き出す）。
- 書き出しフォーマットは WAV（無圧縮）を基準に、gapless メタデータ付き AAC と品質・サイズを
  比較して決める（AAC / MP3 はエンコーダ遅延の無音がループの継ぎ目になりうる。→ [技術検証](./tech-verification.md)）。

### 採用パラメータ（MVP 確定値）

| パラメータ | 値 | 根拠 |
| --- | --- | --- |
| クロスフェード時間 | 30 秒 | 仕様の 30〜60 秒範囲の下限。expo-audio で等パワー実装後に聴感が許容できることを確認 |
| 深まりフェーズ音量 | フル音量の約 48% | 生活音マスクとして十分かつ急な音量変化を起こさない値として採用 |
| フェードイン時間（再生開始） | 4 秒 | 抱っこ中に気づかれず始まる目安 |
| 停止フェード時間 | 0.4 秒 | 停止は即停止。急停止のクリックノイズだけを防ぐ最短フェードとして採用 |
| 試聴自動停止時間 | 18 秒 | 仕様の 15〜20 秒範囲の中間値 |

## 技術リスク

- ギャップレスループ再生と 2 トラック同時再生・クロスフェード。expo-audio で不足する場合は
  ネイティブモジュール（AVAudioEngine / ExoPlayer）を用いる。
- 長いクロスフェード中の音量の谷・位相干渉。等パワークロスフェードで抑える。
